## 摘要
我们研究了功能多数投票（Functional Majority Voting，FMV），这是一种基于功能一致性的代码生成方法，利用大型语言模型在测试输入上的运行时执行特征，从多个生成结果中识别具有代表性的解决方案。我们发现，FMV 作为一种测试时推理策略是有效的，在不显著增加计算开销的情况下，大幅提升了 LiveCodeBench 上的性能。此外，我们扩展了功能一致性的应用，将其作为一种聚合策略用于无标签的测试时强化学习。我们展示了这种方法可以提高在保留任务上的 pass@1 指标，但未发现其能够实现超越基础模型性能上限的自我改进。

## Abstract
We investigate Functional Majority Voting (FMV), a method based on functional consensus for code generation with Large Language Models, which identifies a representative solution from multiple generations using their runtime execution signatures on test inputs. We find that FMV is an effective test-time inference strategy, substantially boosting performance on LiveCodeBench without a large compute overhead. Furthermore, we extend the utility of functional consensus and apply it as an aggregation strategy for label-free Test-Time Reinforcement Learning. We demonstrate that this increases pass@1 on holdout tasks, but find no evidence of self-improvement beyond the base model's performance ceiling.

## 1. 研究背景与核心问题

- **研究动机**  
  大型语言模型（LLM）在代码生成方面虽能生成语法合理的程序，但常出现**推理不稳定、幻觉输出**等问题。如何在**推理阶段（test-time）自动筛选可靠输出**，成为提升模型性能的关键。  
  在数学推理领域已有“自一致性”（self-consistency）等方法，但这些方法依赖语言内容匹配，**不适用于代码生成场景**。然而，代码具有独特的“可执行性”属性——可以通过运行测试来验证结果正确性。

- **研究核心问题**  
  如何利用代码运行行为（execution behavior）在**无标签数据条件下**识别并聚合多重生成结果中的最优方案，从而提升生成质量与泛化性能。

## 2. 方法论：Functional Majority Voting（FMV）

- **核心思想**  
  FMV 基于“功能共识”（functional consensus）：  
  > 不再仅匹配文本输出，而是通过**运行多个候选程序并比较其执行结果一致性**，选择功能表现最稳健的候选程序。

- **主要流程**  
  给定提示 \( P \)，模型生成 \( N \) 个候选程序 \( C = \{c_1, c_2, \dots, c_N\} \)，对一组测试输入 \( X = \{x_1, \dots, x_K\} \)：
  1. **执行阶段**：运行每个候选程序 \( c_i \) 在各输入上，得到输出向量  
     \[
     o_i = [o_{i,1}, o_{i,2}, \dots, o_{i,K}]
     \]
  2. **候选筛选**：去除运行失败（报错、超时、非法格式）的程序。
  3. **功能一致性评分**：
     \[
     S(c_i) = \sum_{j \neq i} \sum_{k=1}^K I(o_{i,k} = o_{j,k})
     \]
     该分数度量候选程序间输出结果的总体一致性。
  4. **最终决策**：选择得分最高的程序 \( c^* = \arg\max S(c_i) \) 作为代表性解。
  
  该过程对应了**软多数投票**逻辑：部分一致即能获得“部分信用”，以避免单一测试差异导致失败。

- **扩展机制：Pointwise-FMV**  
  在无标签强化学习（TTRL）场景中，FMV 的思想可生成“伪标签”。  
  - 对每个测试输入 \( x_k \)，取所有候选输出的众数（mode）作为合成目标 \( y^*_k \)；
  - 构成的合成执行向量 \( y^* \) 可作为强化学习的奖励信号或训练目标。

## 6. 主要结论与发现

- **FMV 可显著提升代码生成性能**：  
  在 LiveCodeBench-v6 上，FMV 提升了平均准确率（Qwen3-4B-Thinking 从 37.7% → 约 52%），优于基线与语义投票。
- **计算效率优势显著**：  
  相比 GenCluster 需要额外模型评判环节，FMV 仅依赖程序执行，成本低。
- **在无标签强化学习中有效**：  
  训练后模型在零样本（zero-shot）任务上准确率提升（31.6% → 34.5%）。
- **但未呈现“自我改进”效应**：  
  训练后的模型未突破基模型性能上限（best@64 反而略降）。
  > 说明 FMV 主要起到“推理结果集成”的作用，而非真正改善模型能力。

## 7. 方法与实验的优点

- **创新性强**：首次将“功能多数投票”系统化研究在代码生成中。  
- **高可解释性**：利用执行一致性衡量结果质量，避免仅依赖文本匹配。  
- **计算开销低**：完全依赖 CPU 执行，不需再调用 LLM。  
- **推广性强**：可直接用于强化学习奖励构建、伪标签生成等下游任务。
