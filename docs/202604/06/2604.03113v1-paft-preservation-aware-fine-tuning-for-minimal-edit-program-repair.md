## 摘要
大型语言模型（LLMs）在自动程序修复中表现出色，但那些能通过完整测试集的可行补丁往往重写了过多代码，从而增加了代码审查和维护成本。这种过度编辑的现象很常见，因为大多数错误是局部化的，而标准的有监督微调并未提供关于哪些标记应被保留、哪些应被修改的显式信号。本文提出 PAFT（Preservation-Aware Fine-Tuning），一种面向保持性的最小编辑程序修复微调方法。PAFT 通过对齐含错误代码与修复后代码来提取标记级保持信号，将其与全序列掩码相结合，并采用编辑难度课程学习。在 Defects4J 和 HumanEval-Java 数据集上，PAFT 相比标准有监督微调（StdFT）可将 pass@1 提升最高达 65.6%，同时将平均编辑距离（AED）减少最高达 32.6%。在基于 DeepSeek-Coder-6.7B 的 Defects4J 上，PAFT 还优于强大的基于偏好的修复基线方法 AdaPatcher，将 pass@1 从 5.9% 提升至 10.1%，并将中位 AED 从 61.0 降低至 42.0。总体而言，PAFT 能够保持稳定上下文，聚焦于错误区域进行编辑，生成更小、更局部化且合理的补丁，无需在推理阶段进行搜索、重排序或后处理。

## Abstract
Large language models (LLMs) are effective for automated program repair, but plausible patches that pass the full test suite often rewrite more code than necessary, increasing review and maintenance costs. This over-editing is common because most bugs are localized, while standard supervised fine-tuning provides no explicit signal about which tokens should be preserved and which should be changed. We propose PAFT, a preservation-aware fine-tuning method for minimal-edit program repair. PAFT derives token-level preservation signals by aligning buggy and fixed code, combines them with full-sequence masking, and applies an edit-difficulty curriculum. Across Defects4J and HumanEval-Java, PAFT improves pass@1 by up to 65.6% over standard supervised fine-tuning (StdFT) while reducing average edit distance (AED) by up to 32.6%. On Defects4J with DeepSeek-Coder-6.7B, PAFT also outperforms AdaPatcher, a strong preference-based repair baseline, improving pass@1 from 5.9% to 10.1% while reducing median AED from 61.0 to 42.0. Overall, PAFT preserves stable context and concentrates edits on faulty regions, yielding smaller, more localized, plausible patches without inference-time search, reranking, or post-processing.

## 一、研究背景与核心问题

- **研究动机**  
  自动化程序修复（Automated Program Repair, APR）借助大语言模型（LLMs）已展现出显著潜力。然而，当前主流方法通常只追求生成能通过测试的“可行补丁”，忽视了补丁的“编辑最小性”。  
  模型往往**重写了过多稳定代码（Over-editing）**，导致：
  - 代码审查代价高；
  - 维护风险增加；
  - 难以判断代码的真实修改意图。

- **核心问题**  
  标准的有监督微调（Supervised Fine-Tuning, SFT）优化模型生成参考修复，但没有显式区分**应保留的稳定上下文**与**应修改的故障区域**。因此，模型缺乏“保留意识”，即“哪些部分应该不变”。

## 二、方法论：PAFT 的核心思想与技术细节

### 1. 核心思想
PAFT（Preservation-Aware Fine-Tuning）旨在让模型具备“保留稳定代码上下文”的能力，同时在修正错误区域时实现最小修改。  
其思路是：直接从配对的缺陷代码与修复代码中提取“保留信号”，并在训练阶段通过加权损失方式奖励模型复制稳定代码。

### 2. 三大关键机制
1. **Token-level 对齐保留信号**
   - 利用 Ratcliff/Obershelp 模式匹配算法识别缺陷代码与修复代码的**稳定匹配块**；
   - 对齐结果生成一个二进制 mask，指示哪些 token 应被保留；
   - 对这些“匹配 token”在训练损失中施加权重提升（\(w_{\text{align}}>1\)）。

2. **全序列掩码（Full-sequence Masking）**
   - 相较于仅计算响应部分损失（assistant-only masking），PAFT 在整条输入序列上优化损失；
   - 使模型能同时学习上下文、指令与代码生成联系，减少信息割裂。

3. **基于编辑难度的课程学习（Edit-difficulty Curriculum）**
   - 使用**编辑行数**作为难度度量（新增行+删除行总数）；
   - 模型先学习小规模改动样例，再逐步过渡到大型修改；
   - 实现稳定的训练收敛。

### 3. 训练公式概述
对于每个 token \( x_t \)，PAFT 的加权损失函数为：

\[
L(\theta) = \frac{\sum_t M_t w_t \, (-\log p_\theta(x_t \mid x_{<t}))}{\sum_t M_t w_t}
\]
其中：
- \(M_t\)：掩码；
- \(w_t\)：若 token 属于对齐稳定区域，则 \( w_t = w_{align} > 1 \)。

通过此机制，模型在训练时偏向“复制稳定内容”，减少无故重写。
