## 摘要
大型语言模型（LLMs）在程序修复任务上表现优异，但常常遭遇过度编辑的问题，即过多的修改覆盖了正确代码，从而妨碍了错误定位。我们系统地量化了这种现象的影响，并提出了精确修复任务，其目标是在最大化正确代码复用的同时，只修复存在缺陷的部分。基于这一洞察，我们提出了 PRepair 框架，该框架可缓解过度编辑并提高修复准确率。PRepair 包含两个组件：自破坏（Self-Breaking），通过受控的缺陷注入与极值采样生成多样化的错误程序；以及自修复（Self-Repairing），利用编辑感知的群体相对策略优化（Edit-Aware Group Relative Policy Optimization，EA-GRPO）训练模型，并通过编辑感知奖励引导其进行最小且正确的编辑。实验结果表明，PRepair 在 $\mathrm{fix}_1@1$ 指标下的修复精度最多提高 31.4%，该指标同时考虑修复正确性与修改范围。当结合推测编辑（speculative editing）使用时，PRepair 还显著提升了解码吞吐量，展示了其在精确且实用的代码修复方面的潜力。

## Abstract
Large Language Models (LLMs) achieve strong program repair performance but often suffer from over-editing, where excessive modifications overwrite correct code and hinder bug localization. We systematically quantify its impact and introduce precise repair task, which maximizes reuse of correct code while fixing only buggy parts. Building on this insight, we propose PRepair, a framework that mitigates over-editing and improves repair accuracy. PRepair has two components: Self-Breaking, which generates diverse buggy programs via controlled bug injection and min-max sampling, and Self-Repairing, which trains models with Edit-Aware Group Relative Policy Optimization (EA-GRPO) using an edit-aware reward to encourage minimal yet correct edits. Experiments show that PRepair improves repair precision by up to 31.4% under $\mathrm{fix}_1@1$, a metric that jointly considers repair correctness and extent, and significantly increases decoding throughput when combined with speculative editing, demonstrating its potential for precise and practical code repair.

## 二、方法论：PRepair 框架与 EA-GRPO 策略

### 2.1 核心思想
通过引入“编辑感知”的奖励机制，使模型在追求正确性的同时控制编辑范围，实现“**少改但改对**”的修复策略。  
整体框架包括两个阶段：

1. **Self-Breaking（自破坏）**：  
   生成具有多样化错误模式的训练样本；
2. **Self-Repairing（自修复）**：  
   采用强化学习优化策略，在奖励函数中同时考虑修复正确率与编辑代价。

---

### 2.2 方法结构与关键机制

#### （1）框架总览
PRepair 包含两个阶段（见 Figure 3）：

- **① 自破坏（Self-Breaking）**  
  - 模型接收正确代码后，通过**受控的缺陷注入（bug injection）**自动生成多个错误版本。  
  - 使用“**极小化最大相似度采样（min–max sampling）**”策略筛选样本：
    \[
    X_s = \min_{X' \subset X} \max_{X_i,X_j \in X'} (1 - DEC(X_i,X_j))
    \]
    以保持错误样本之间的多样性。

- **② 自修复（Self-Repairing）**  
  - 训练模型使用**EA-GRPO（Edit-Aware Group Relative Policy Optimization）**强化学习算法。  
  - 奖励函数结合正确率与编辑代价（Edit Cost，基于行级Levenshtein距离）：
    \[
    DEC(X, Y) = \frac{D(X, Y)}{|X|}
    \]
  - 当修复组的平均正确率超过阈值 α 时，引入动态编辑惩罚 β，以防止过度修改：
    \[
    R_i^G = 
    \begin{cases}
    1 - T(G)\cdot \beta \cdot P_i^G, & \text{若样本正确;}\\
    0, & \text{否则。}
    \end{cases}
    \]
  - 该方法无需参考正确样例，仅通过“输入错误代码与输出结果的编辑距离”计算奖励。

#### （2）指标创新：fix p@k  
提出新的评估指标 **fix p@k**，综合修复正确性和修改范围：
\[
c = \sum_{i=1}^{n} I(\text{correct}_i \land \frac{DEC(X,Y'_i)}{DEC(X,Y)} \le p)
\]
其中 p 为允许的编辑开销比。该指标比传统的 pass@k 更能反映“精确修复”能力。

#### （3）推测编辑（Speculative Edits）  
- 应用基于 **Prompt Lookup Decoding** 的推测解码技术，提高推理效率。  
- 理论分析表明，减少编辑距离 DEC 会显著提升吞吐率：
  \[
  T \propto \frac{1 - (1 - DEC)^{K+1}}{DEC}
  \]
  即“编辑越少 → 解码越快”。
