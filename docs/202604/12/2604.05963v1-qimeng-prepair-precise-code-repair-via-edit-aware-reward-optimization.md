## 摘要
大型语言模型（LLMs）在程序修复任务中表现出强大的能力，但常常遭受过度编辑的问题，即过多的修改会覆盖正确的代码，从而妨碍错误定位。我们系统地量化了这一影响，并提出了精确修复任务，该任务旨在最大化正确代码的复用，同时仅修复存在错误的部分。基于这一洞察，我们提出了 PRepair 框架，用于缓解过度编辑并提高修复准确性。PRepair 包含两个组件：Self-Breaking，通过可控的错误注入和极值采样来生成多样化的错误程序；Self-Repairing，使用 Edit-Aware Group Relative Policy Optimization（EA-GRPO）进行训练，并引入基于编辑的奖励机制以鼓励最小且正确的编辑。实验结果表明，PRepair 在 $\mathrm{fix}_1@1$ 指标下的修复精度提高了最多 31.4%，该指标同时考虑修复的正确性和编辑范围；此外，当与推测编辑结合使用时，显著提升了解码吞吐量，展示了其在精确且实用的代码修复方面的潜力。

## Abstract
Large Language Models (LLMs) achieve strong program repair performance but often suffer from over-editing, where excessive modifications overwrite correct code and hinder bug localization. We systematically quantify its impact and introduce precise repair task, which maximizes reuse of correct code while fixing only buggy parts. Building on this insight, we propose PRepair, a framework that mitigates over-editing and improves repair accuracy. PRepair has two components: Self-Breaking, which generates diverse buggy programs via controlled bug injection and min-max sampling, and Self-Repairing, which trains models with Edit-Aware Group Relative Policy Optimization (EA-GRPO) using an edit-aware reward to encourage minimal yet correct edits. Experiments show that PRepair improves repair precision by up to 31.4% under $\mathrm{fix}_1@1$, a metric that jointly considers repair correctness and extent, and significantly increases decoding throughput when combined with speculative editing, demonstrating its potential for precise and practical code repair.

## 一、研究动机与背景

- **核心问题**：当前大型语言模型（LLMs）在程序修复任务中虽然能生成正确功能代码，但存在严重的“**过度编辑（over-editing）**”问题——它们倾向于重写大段代码，错误地覆盖原本正确的部分，不仅降低修复准确率，也增加开发者代码审查负担。  
- **研究意义**：论文提出“**精确修复任务（Precise Repair）**”，希望在保持正确逻辑的前提下，仅对存在错误的部分进行最小化修改，以提高修复质量与实用性。  
- **主要矛盾**：现有方法大多只优化修复正确率（correctness），忽视编辑范围（extent of modifications）的影响；因此，即使代码功能正确，也可能因编辑范围过大导致维护性下降。

## 二、方法论：PRepair 框架与 EA-GRPO 优化

### 1. 总体思想
论文提出 **PRepair**（Precise Repair）框架，由两大阶段组成：
- **Self-Breaking（自我破坏）**：自动生成高质量的“带错误代码”训练数据；
- **Self-Repairing（自我修复）**：通过强化学习优化模型，使其在修复时保持最小编辑。

核心创新：引入 **编辑感知奖励优化（Edit-Aware GRPO, EA-GRPO）**，将代码修复正确性与编辑代价联合建模，形成动态奖励机制。

### 2. 关键技术与算法机制

#### （1）编辑成本定义（Edit Cost）
- 基于 **Levenshtein 距离**，计算由错误代码 \( X \) 到修复后代码 \( Y \) 的最小插入、删除、替换数。
- 定义归一化编辑成本：
  \[
  DEC(X, Y) = \frac{D(X, Y)}{|X|}
  \]
  其中 |X| 为原代码行数，用于不同程序间比较。

#### （2）指标设计：fix p@k
- 提出新的评价指标 **fix p@k**，综合考虑正确性（是否通过测试）与编辑范围（编辑成本不超过阈值比例）。  
- 与传统 pass@k 相比，更能反映“精确修复”能力。

#### （3）EA-GRPO 奖励机制
- 基于 **Group Relative Policy Optimization (GRPO)** 框架扩展。
- 动态奖励结构：
  - **分组准确率阈值 (α)** 控制何时施加编辑惩罚；
  - 仅在“达到足够正确率”后启用惩罚机制；
  - 奖励函数形式：
    \[
    R^G_i = 
    \begin{cases}
    1 - T(G) \cdot β \cdot P_i^G, & \text{若样本正确}\\
    0, & \text{否则}
    \end{cases}
    \]
    其中 \( P_i^G \) 通过标准化编辑成本计算，β 为惩罚系数。

#### （4）自我破坏策略（Self-Breaking）
- 模型接收正确代码及任务描述，自动注入错误（通过 prompt 控制），形成大规模“带 bug 数据集”。
- 为保证多样性，采用 **min–max 采样策略**：最小化所有样本间的编辑相似度，避免集中于单一错误模式。

#### （5）推测编辑（Speculative Edits）
- 利用 **Prompt Lookup Decoding** 实现高速推测生成，通过输入提示的 n-gram 重用机制并行验证。
- 理论上推导 **解码吞吐量与编辑成本的反比关系**：编辑越少，推理速度越快。
