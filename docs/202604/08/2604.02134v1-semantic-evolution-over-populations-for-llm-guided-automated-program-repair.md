## 摘要
大型语言模型（LLMs）近年来在自动程序修复（APR）领域展现出强大的潜力，尤其是在通过迭代优化生成和改进候选补丁方面。然而，现有最先进的基于LLM的迭代优化APR方法仍难以充分应对多项挑战，包括保持修复假设间的有益多样性、识别语义相关的修复族、组合互补的部分修复、利用结构化故障信息以及摆脱结构上有缺陷的搜索区域。本文提出了一种面向APR迭代优化的人口语义进化框架，称为EvolRepair，该框架将基于LLM的APR形式化为语义进化算法。EvolRepair重新定义了经典遗传算法在APR中的搜索范式，但用由LLM和结构化执行反馈驱动的语义感知组件取代了基于语法的操作符。候选修复被组织为行为一致的群组，使算法能够维持多样性、在修复家族间进行推理，并通过在群体间重组互补的修复洞见来合成更优的候选方案。通过利用结构化故障模式引导搜索方向，EvolRepair既能优化有潜力的修复策略，又能在必要时转向其他抽象层次。实验结果表明，EvolRepair在修复效果上显著优于现有的基于LLM的APR方法。

## Abstract
Large language models (LLMs) have recently shown strong potential for automated program repair (APR), particularly through iterative refinement that generates and improves candidate patches. However, state-of-the-art iterative refinement LLM-based APR approaches cannot fully address challenges, including maintaining useful diversity among repair hypotheses, identifying semantically related repair families, composing complementary partial fixes, exploiting structured failure information, and escaping structurally flawed search regions. In this paper, we propose a Population-Based Semantic Evolution framework for APR iterative refinement, called EvolRepair, that formulates LLM-based APR as a semantic evolutionary algorithm. EvolRepair reformulates the search paradigm of classic genetic algorithm for APR, but replaces its syntax-based operators with semantics-aware components powered by LLMs and structured execution feedback. Candidate repairs are organized into behaviorally coherent groups, enabling the algorithm to preserve diversity, reason over repair families, and synthesize stronger candidates by recombining complementary repair insights across the population. By leveraging structured failure patterns to guide search direction, EvolRepair can both refine promising repair strategies and shift toward alternative abstractions when necessary. Our experiments show that EvolRepair substantially improves repair effectiveness over existing LLM-based APR approaches.

## 一、研究背景与核心问题

- **研究背景**  
  自动程序修复（Automated Program Repair, APR）旨在自动生成代码修复补丁，使程序通过所有测试用例。传统方法多基于模板、语法变异或搜索优化，但常陷入语义过拟合、搜索空间有限等问题。  
  近年来，大型语言模型（LLM）在 APR 领域表现突出，能通过生成式推理和上下文理解自动提出补丁。然而，LLM 的**迭代修复方法**依旧存在以下瓶颈：
  1. 修复假设多样性不足，容易陷入单一修复方向；
  2. 缺乏对“语义相关修复族”（即不同补丁间语义等价类）的识别；
  3. 难以组合不同候选补丁中的**互补的部分修复**；
  4. 常依赖测试通过率（scalar fitness）而未充分利用结构化失败信息；
  5. 一旦当前修复方向错误，难以“抽象层切换”逃离局部最优。
  
- **核心问题**  
  如何在 LLM 引导下实现一种**语义驱动、群体化的搜索策略**，既能保持候选修补方案的多样性，又能利用语义信号聚合、重组和优化补丁，使修复更高效且语义更正确。

## 二、方法论与算法框架

### 1. 核心思想：语义进化算法（Semantic Evolutionary Algorithm）

- 将基于 LLM 的程序修复**重新定义为语义进化搜索**，即：
  - 借鉴遗传算法（GA）“多候选体、进化迭代”的思想；
  - 但抛弃GA中的语法级操作符（如随机突变、文本拼接）；
  - 用“语义感知的LLM算子”替代，实现更智能的修复生成。

### 2. 框架总览（EvolRepair）

![](assets/figures/arxiv/2604.02134v1/fig-007.webp)

- EvolRepair包含五个关键要素：
  1. **语义变异（Semantic Mutation）**：  
     LLM根据失败用例、错误栈等结构化反馈，对个体候选代码进行定向修复改写。
  2. **行为分组（Behavioral Grouping）**：  
     根据测试通过集（passed test set）对候选补丁聚类，形成“语义相似群族”。
  3. **跨群重组（Population-Level Recombination）**：  
     由LLM汇总多个候选补丁的代码逻辑，综合优点、修补弱点，实现语义级交叉融合。
  4. **选择与生存（Survivor Selection）**：  
     去重后按测试覆盖率、通过率、代码长度排序，筛选出Top-N进入下一代。
  5. **自适应抽象跳跃（Adaptive Abstraction Shift）**：  
     当特定群族修复失败时，算法根据失效模式切换到新的抽象层重新搜索。

### 3. 算法流程概述（Algorithm 1）

1. **初始化**：调用LLM生成初始候选补丁种群；
2. **评估**：运行测试用例，得到行为签名（behavioral signature）；
3. **分组与重组**：基于Jaccard相似度对行为分组；再进行跨群体采样与融合；
4. **语义变异**：基于失败反馈提示LLM微调候选；
5. **筛选下一代**：按适应度（通过率+多样性）选择；
6. **迭代数轮**直到产生完整修复。
