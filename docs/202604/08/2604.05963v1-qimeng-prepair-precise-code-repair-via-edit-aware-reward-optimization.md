## 摘要
大型语言模型（LLMs）在程序修复任务中表现出色，但常常受到过度编辑（over-editing）问题的影响——即过多的修改覆盖了原本正确的代码，从而妨碍了错误定位。我们系统地量化了这种现象的影响，并提出了“精确修复（precise repair）”任务，其目标是在修复错误部分的同时最大限度地复用正确代码。基于这一洞察，我们提出了 PRepair 框架，用于缓解过度编辑问题并提升修复精度。PRepair 由两个核心组件组成：Self-Breaking，通过受控错误注入和极大极小采样（min-max sampling）生成多样化的错误程序；Self-Repairing，通过编辑感知分组相对策略优化（Edit-Aware Group Relative Policy Optimization, EA-GRPO）训练模型，利用“编辑感知奖励（edit-aware reward）”鼓励最小且正确的编辑。实验结果显示，PRepair 在 $\mathrm{fix}_1@1$ 指标（同时考虑修复正确率与修改范围）下可将修复精度提升最多 31.4%，并在结合推测性编辑（speculative editing）时显著提高了解码吞吐量，充分展示了其在精确且实用的代码修复中的潜力。

## Abstract
Large Language Models (LLMs) achieve strong program repair performance but often suffer from over-editing, where excessive modifications overwrite correct code and hinder bug localization. We systematically quantify its impact and introduce precise repair task, which maximizes reuse of correct code while fixing only buggy parts. Building on this insight, we propose PRepair, a framework that mitigates over-editing and improves repair accuracy. PRepair has two components: Self-Breaking, which generates diverse buggy programs via controlled bug injection and min-max sampling, and Self-Repairing, which trains models with Edit-Aware Group Relative Policy Optimization (EA-GRPO) using an edit-aware reward to encourage minimal yet correct edits. Experiments show that PRepair improves repair precision by up to 31.4% under $\mathrm{fix}_1@1$, a metric that jointly considers repair correctness and extent, and significantly increases decoding throughput when combined with speculative editing, demonstrating its potential for precise and practical code repair.

## 一、研究动机与核心问题

- **背景**：随着大语言模型（LLMs）被广泛应用于程序修复任务，其在自动修复错误代码方面表现优异，但仍面临“**过度编辑（over-editing）**”现象。  
  - 模型常常在修复过程中进行大量非必要的修改，覆盖掉原本正确的代码。
  - 这种问题导致**错误定位困难**、**代码可维护性降低**、并增加开发者的复审负担。

- **研究核心问题**：如何让模型在修复过程中做到“**精准编辑**”，即：
  - 最大限度地复用原始正确逻辑；
  - 仅修改真正错误的部分；
  - 保证修复正确且改动最小。

- **研究目标**：作者提出了新的“**Precise Repair（精确修复）**”任务，并针对现有 LLM 训练过程中过度关注修复正确性、忽略编辑范围的弊端，设计了新的奖励优化机制与数据生成框架。

## 二、方法与核心技术

### 2.1 总体框架：PRepair
- **框架组成**：包括两个主要阶段：
  1. **Self-Breaking（自我破坏）** —— 生成多样化的错误代码；
  2. **Self-Repairing（自我修复）** —— 通过带编辑感知奖励的强化学习优化模型。

### 2.2 Self-Breaking：数据生成策略
- 目标：构建高质量且多样的系统性错误数据，弥补真实含局部错误的程序稀缺问题。
- 方法：
  - 利用模型自主在正确代码中**注入错误（bug injection）**；
  - 结合**Min–Max采样策略**，即在一批样本中选择那些差异最大的错误版本，以保证多样性；
  - 计算样本间的编辑距离 DEC(X, Y)，并最小化样本间相似度。

### 2.3 Self-Repairing：模型训练策略
- 使用强化学习框架 **GRPO（Group Relative Policy Optimization）** 的改进版：
  - 提出 **EA-GRPO（Edit-Aware GRPO）**，核心思想是在正确性奖励之外引入“**编辑感知奖励（edit-aware reward）**”。
  
- **公式机制说明**：
  - 定义归一化编辑成本：  
    $$ DEC(X, Y) = \frac{D(X, Y)}{|X|} $$
    其中 \( D(X, Y) \) 为 Levenshtein 距离，表示从错误代码到修复代码的最小编辑数。
  - 动态奖励设计：
    - 当一组生成结果的正确率达到阈值 α 时，才施加编辑惩罚；
    - 奖励公式：  
      $$ R_i^G = 1 - T(G)·β·P_i^G $$
      其中 β 为惩罚系数，P 表示标准化编辑成本。
  - 这种动态奖励使模型在正确修复的前提下，倾向于最小修改。

### 2.4 新评价指标：Fix p@k
- 为同时衡量**修复正确性**和**修改范围**，作者提出新指标：
  - fix p@k = 同时通过测试且编辑量在容许比例 p 内的样本占比；
  - 其中 p 表示编辑容忍度，k 表示候选数量。

### 2.5 推测性编辑（Speculative Edits）
- 结合推测性解码（Prompt Lookup Decoding）技术，加速推理：
  - 理论上推测编辑的吞吐量与编辑成本成反比；
  - 当编辑量减少时，吞吐量显著提升。
