## 摘要
大型语言模型（LLMs）在自动程序修复中表现出较高的有效性，但那些能够通过完整测试集的可行补丁往往会修改比必要更多的代码，从而增加代码审查和维护成本。这种过度编辑的现象十分常见，因为大多数缺陷是局部化的，而标准的监督微调方法并未提供关于哪些标记应被保留、哪些应被修改的明确信号。我们提出了PAFT，一种面向最小编辑程序修复的保留感知微调方法。PAFT通过对比缺陷代码与修复后代码来生成标记级别的保留信号，将其与全序列掩码结合，并应用基于编辑难度的课程学习。在Defects4J和HumanEval-Java数据集上，与标准监督微调（StdFT）相比，PAFT可使pass@1提升最高达65.6%，同时将平均编辑距离（AED）降低最多32.6%。在使用DeepSeek-Coder-6.7B的Defects4J上，PAFT还优于强大的基于偏好的修复基线方法AdaPatcher，使pass@1从5.9%提升至10.1%，并将中位数AED从61.0降至42.0。总体而言，PAFT能够保留稳定的上下文，并将编辑集中于错误区域，在无需推理阶段搜索、重排序或后处理的情况下，生成更小、更局部化且更合理的补丁。

## Abstract
Large language models (LLMs) are effective for automated program repair, but plausible patches that pass the full test suite often rewrite more code than necessary, increasing review and maintenance costs. This over-editing is common because most bugs are localized, while standard supervised fine-tuning provides no explicit signal about which tokens should be preserved and which should be changed. We propose PAFT, a preservation-aware fine-tuning method for minimal-edit program repair. PAFT derives token-level preservation signals by aligning buggy and fixed code, combines them with full-sequence masking, and applies an edit-difficulty curriculum. Across Defects4J and HumanEval-Java, PAFT improves pass@1 by up to 65.6% over standard supervised fine-tuning (StdFT) while reducing average edit distance (AED) by up to 32.6%. On Defects4J with DeepSeek-Coder-6.7B, PAFT also outperforms AdaPatcher, a strong preference-based repair baseline, improving pass@1 from 5.9% to 10.1% while reducing median AED from 61.0 to 42.0. Overall, PAFT preserves stable context and concentrates edits on faulty regions, yielding smaller, more localized, plausible patches without inference-time search, reranking, or post-processing.

### 背景

### 问题核心
- 现有的监督微调（SFT）仅优化模型重现参考补丁的能力，缺乏：
  - 对哪些代码 **应保留**、哪些 **应修改** 的显式指导；
  - 控制修改范围的机制。
- 因此论文提出一个关键研究问题：  
  **能否在保持或提升修复正确率的同时，减少不必要的代码修改？**

### 研究目标
- 设计一种新的微调机制，使模型在修复缺陷时更加**局部化（localized）**，即：
  - **保留稳定上下文（stable context）**；
  - **仅在缺陷区域修改**；
  - **不依赖推理阶段的搜索、重排序或后处理。**

## 2. 方法论：PAFT（Preservation-Aware Fine-Tuning）

### 核心思想
PAFT在训练阶段引入「保留感知」信号，使模型显式学习哪些 Token 应该保持不变。  
主要创新点在于：
1. **通过源码对齐生成token级保留信号**；
2. **在损失函数中动态调整token权重**；
3. **结合全序列掩码与编辑难度课程学习** 实现稳定的优化。

### 核心组成

#### (1) 对齐信号生成
- 将“缺陷代码 (b)”与“修复代码 (f)”进行**token级序列对齐**。  
- 使用 **Ratcliff/Obershelp 匹配算法** 识别连续匹配区间。  
- 对齐结果生成一个二值掩码 `I_align`：  
  - 对齐部分视为应“保留”的 token；  
  - 未对齐部分为“候选修改”区域。

#### (2) 保留感知的加权损失
- 对每个 token 的负对数似然损失 `ℓ_t(θ)` 增加权重：
  \[
  w_t = 
  \begin{cases} 
    w_{align} & t \in I_{align} \\
    1 & \text{otherwise}
  \end{cases}
  \]
- 默认 `w_align = 2.0`，即强化模型在稳定token上的学习。  
- 总损失为加权Masked NLL：
  \[
  L(\theta; x, M) = \frac{\sum_t M_t w_t \ell_t(\theta)}{\sum_t M_t w_t}
  \]
- 采用 **QLoRA (4-bit NF4)** 微调，以低显存训练大模型。

#### (3) 编辑难度课程（Curriculum）
- 按行级差异（`diff lines`）从小到大排序训练样本。  
- 使模型先学习简单的、小范围修改，再逐步适应复杂修复。

#### (4) 全序列掩码
- 与传统仅在“回答部分”计算损失的SFT不同，PAFT在**完整输入-输出序列上计算loss**，从而增强上下文保持能力。

#### (5) 推理阶段
- 推理时无需额外步骤（无搜索、无重排序、无权重调整），直接自左向右生成补丁。
