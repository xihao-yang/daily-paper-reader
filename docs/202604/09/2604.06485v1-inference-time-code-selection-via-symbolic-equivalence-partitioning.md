## 摘要
“Best-of-N” 选择是一种在使用大型语言模型（LLMs）进行代码生成时常用的推理阶段扩展方法。然而，为了可靠地识别正确的解决方案，现有方法往往依赖于昂贵或随机性的外部验证器。本文提出了符号等价划分（Symbolic Equivalence Partitioning），这是一种利用符号执行将候选程序按语义行为分组，并从主导功能划分中选取代表程序的选择框架。为改进分组和选择，我们在符号执行过程中将特定领域的约束编码为可满足性模理论（Satisfiability Modulo Theories，SMT）假设，以减少路径爆炸并防止在问题域之外进行无效输入搜索。在 N=10 时，我们的方法在无需额外 LLM 推理（超出初始 N 个候选生成）的情况下，将 HumanEval+ 上的平均准确率从 0.728 提升至 0.803，LiveCodeBench 上从 0.516 提升至 0.604。

## Abstract
"Best-of-N" selection is a popular inference-time scaling method for code generation using Large Language Models (LLMs). However, to reliably identify correct solutions, existing methods often depend on expensive or stochastic external verifiers. In this paper, we propose Symbolic Equivalence Partitioning, a selection framework that uses symbolic execution to group candidate programs by semantic behavior and select a representative from the dominant functional partition. To improve grouping and selection, we encode domain-specific constraints as Satisfiability Modulo Theories (SMT) assumptions during symbolic execution to reduce path explosion and prevent invalid input searches outside the problem domain. At N=10, our method improves average accuracy over Pass@1 from 0.728 to 0.803 on HumanEval+ and from 0.516 to 0.604 on LiveCodeBench, without requiring any additional LLM inference beyond the initial N candidate generations.

## 二、方法论：符号等价划分（Symbolic Equivalence Partitioning, SEP）

### 1. 核心思想  
- 将 N 个 LLM 生成的候选程序通过**符号执行（symbolic execution）**进行语义分析，判断程序间在问题域内的功能等价性。  
- 基于“语义等价”定义构建**等价类划分（equivalence partitioning）**，选择其中占比最大（即语义最一致）的功能簇代表作为最终输出。  
- 理论假设：正确解在高质量候选集中往往形成“多数派”，具有一致语义行为；错误解则表现出分散的功能差异。

### 2. 关键技术要点  

#### (1) 符号等价判定公式  
若程序 \(p_i, p_j\) 在约束域 \(C\) 下对所有输入 \(x \in Dom(C)\) 的执行结果均相同，则有：  
\[
p_i \equiv_C p_j \iff \forall x, EO_i(x) = EO_j(x)
\]  
其中 \(EO_k(x)\) 为程序输出、异常及状态变化的综合执行结果。  

等价判定转化为可满足性问题：  
\[
\exists x \text{ s.t. } C(x) \wedge PC(x) \wedge (EO_i(x) \neq EO_j(x))
\]  
若SMT求解器找不到可行解，则两程序视为等价。

#### (2) SMT约束剪枝（SMT-Constrained Pruning）  
- 提出关键技术以避免符号执行中的“路径爆炸（path explosion）”：
  - **从任务描述中提取输入约束**（如数据类型、输入区间等），并以SMT假设的方式注入求解器，而非程序级assert。
  - 若求解路径超出合法输入域，执行器直接忽略该分支，显著提高分析效率与稳健性。

#### (3) 贪心代表划分算法（Greedy Representative Partitioning）  
- 逐个处理候选程序，与已有聚类代表进行等价性判定。  
- 若与某个现有簇代表等价，则加入该簇；否则新建簇。  
- 最终选择**最大簇的代表程序**作为最终输出。  
- 时间复杂度优化至 \(O(N \cdot M)\)（M为聚类数，远小于N），并利用公共样例I/O进行预先筛选以减少符号比较次数。

## 三、实验设计

### 1. 数据集与任务场景  
- **HumanEval+**：164个Python函数生成任务，数据较原始HumanEval扩大80倍隐测规模，用于更严格的验证。
- **LiveCodeBench**：新引入的动态竞赛级代码生成基准，收录438个LeetCode/Codeforces等平台的新题，难度更高、污染更少。

### 2. 对比基线  
- **Pass@1 / Pass@N**：代表基础模型性能和最优上限。  
- **CodeT**（Chen et al., 2023）：通过自生成测试用例评估候选正确性。  
- **LLM-Judge**：生成器本身充当验证器自我评估。  
- **CodeBLEU** 与 **Embedding Consensus**：基于语法/语义相似度的聚类方法。

### 3. 模型范围  
共测试 8 种 LLM（4B–20B 参数），包括 Phi-4、Qwen-3、Llama-3.1-8B、GPT-OSS-20B 等，涵盖指令调优与推理专用版本，确保广泛性。

### 4. 计算环境  
- **生成**：GPU (NVIDIA H100)  
- **符号执行与验证**：CPU (AMD Ryzen 7 7700X)  
- 符号分析在每个候选对上设定固定的时间与路径预算以限制计算规模。  
- 无模型微调或额外训练；SEP仅作用于推理阶段。

## 四、实验结果与充分性

### 1. 精度提升  
在默认 \(N=10\) 设置下：
- **HumanEval+**：平均准确率由 0.728→0.803。  
- **LiveCodeBench**：由 0.516→0.604。  
两者均超过最强基线（CodeT, LLM-Judge）的平均水平。

### 2. 扩展实验  
- 不同样本数量（N=5,10,20）下SEP持续稳定增益，表现优于所有非oracle方法。  
- 对比测试数共达48组（8模型 × 2数据集 × 3采样规模），在其中46组中SEP优胜或并列第一。  
- **Pairwise等价判定准确率**平均0.898，说明符号分析的近似判定在实践中可靠。

### 3. 成本分析  
- SEP整体壁钟时间随N线性增加：HumanEval+ (511s→2559s)，LiveCodeBench (989s→4779s)。  
- 与CodeT相比，SEP避免了额外生成测试样例的巨大GPU代价（CodeT最高需数小时至十余小时）。  
- 表明SEP计算代价**可控且效率更高**。

### 4. 公平性与客观性  
- 使用各模型官方推荐采样参数、未做超参调优。  
- 公布完整模型配置与资源统计，评估标准一致，具备良好可复现性。

## 五、主要结论与发现

- 提出一种**无需额外推理或验证器的推理阶段代码选择框架**。

- **无需额外推理**：相较于CodeT等方法，无额外测试或验证推理步骤。
- **广泛验证**：涵盖8个不同规模与类型的模型，两个标准基准，充分展示方法普适性。

## 七、不足与局限

- **语言依赖性**：当前实现基于Python生态（CrossHair + Z3）；迁移至C++/Java等语言需额外工具开发。  
- **符号执行局限**：分析受限于求解器预算，未能覆盖深递归或特定输入触发逻辑。  
- **扩展性问题**：复杂度约为 \(O(N·M)\)，在大规模候选集下仍可能成为瓶颈。  
- **隐式约束提取不完善**：自然语言描述中未明确的输入条件可能造成语义误判。  
- **实时性待改进**：虽低于CodeT成本，但对于在线应用场景仍需进一步优化。
