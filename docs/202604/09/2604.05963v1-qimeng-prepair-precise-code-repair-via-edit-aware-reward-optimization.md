## 摘要
大型语言模型（LLMs）在程序修复任务中表现出强劲的性能，但常常存在过度编辑（over-editing）的问题，即对代码进行过多不必要的修改，从而覆盖了原本正确的部分并妨碍了错误定位。我们系统地量化了这一现象的影响，并提出了精准修复任务，其目标是在修复错误部分的同时最大限度地重用正确代码。基于这一洞见，我们提出了 PRepair 框架，用于缓解过度编辑并提升修复精度。PRepair 包含两个组件：Self-Breaking 组件通过受控漏洞注入与极大极小采样（min-max sampling）生成多样化的错误程序；Self-Repairing 组件利用带有编辑感知奖励的编辑感知分组相对策略优化（Edit-Aware Group Relative Policy Optimization, EA-GRPO）对模型进行训练，鼓励其进行最小但正确的编辑。实验结果表明，PRepair 在 $\mathrm{fix}_1@1$ 指标下的修复精度提高了高达 31.4%，该指标同时考虑了修复的正确性和修改程度。当与推测式编辑（speculative editing）结合时，PRepair 显著提升了解码吞吐量，展示了其在精准且实用的代码修复方面的潜力。

## Abstract
Large Language Models (LLMs) achieve strong program repair performance but often suffer from over-editing, where excessive modifications overwrite correct code and hinder bug localization. We systematically quantify its impact and introduce precise repair task, which maximizes reuse of correct code while fixing only buggy parts. Building on this insight, we propose PRepair, a framework that mitigates over-editing and improves repair accuracy. PRepair has two components: Self-Breaking, which generates diverse buggy programs via controlled bug injection and min-max sampling, and Self-Repairing, which trains models with Edit-Aware Group Relative Policy Optimization (EA-GRPO) using an edit-aware reward to encourage minimal yet correct edits. Experiments show that PRepair improves repair precision by up to 31.4% under $\mathrm{fix}_1@1$, a metric that jointly considers repair correctness and extent, and significantly increases decoding throughput when combined with speculative editing, demonstrating its potential for precise and practical code repair.

## 一、研究背景与核心问题

- **问题背景**  
  大型语言模型（LLMs）在自动程序修复（Program Repair）中展现出强大能力，但普遍存在**过度编辑（over-editing）**问题。即模型倾向于对整个程序大幅重写，而非精准定位并修复错误部分，这导致：
  - 正确代码被覆盖；
  - 错误定位能力下降；
  - 开发者的代码审核成本增加；
  - 生成代码的维护性降低。

- **研究动机**  
  当前多数方法仅优化“修复是否正确”，忽略“修改程度”，使模型无法学会“少改但改对”的精准策略。因此，作者提出“**精准代码修复（precise repair）**”这一新任务：在保持正确性的同时，最大限度重用原始正确代码，仅修复必要部分。

## 二、方法论：PRepair 框架

### 1. 核心思想

PRepair旨在改进LLM的代码修复精度，通过**自构造缺陷数据**和**编辑感知强化学习优化**，实现“最小修改且正确修复”的目标。  
整个框架包含两个阶段：

- **Self-Breaking（自破坏阶段）**：模型主动生成高质量、有多样性的“错误代码”；
- **Self-Repairing（自修复阶段）**：模型在这些错误样本上进行训练，通过“编辑感知奖励”优化修复策略。

### 2. 模型机制：EA-GRPO算法

在传统的GRPO（Group Relative Policy Optimization）基础上提出**EA-GRPO（Edit-Aware GRPO）**，关键创新如下：

- **编辑感知奖励（Edit-Aware Reward）**：
  - 奖励项同时考虑代码修复的正确性与修改范围；
  - 当修复准确度达到阈值 α 时才施加编辑惩罚；
  - 惩罚值依据当前样本的编辑距离标准化计算（基于Levenshtein距离）。
  - 公式简述：
    - 编辑成本：  
      \( DEC(X,Y) = D(X,Y)/|X| \)，反映修改比例；
    - 奖励定义：  
      若样本正确，\( R = 1 - T(G)·β·P_G \)，其中 \(T(G)\) 控制是否应用惩罚，β为惩罚强度。

- **min–max采样策略**（Self-Breaking阶段）：
  - 为生成多样化的错误样本，模型通过最小化最大相似度挑选错误样本，避免集中于单一错误类型。

### 3. 新评价指标：fix p@k

- 结合**正确性**和**编辑程度**的综合指标：  
  当代码修复正确且编辑成本不超过设定比例时计为成功。  
  - 例如 fix₁@₁ 表示修复正确且修改量 ≤ 原始编辑成本的 1 倍。

### 4. 运行加速：Speculative Edits（推测式编辑）

- 利用**Prompt Lookup Decoding**实现高效推理：
  - 通过减少编辑距离，提升草稿令牌接受率；
  - 理论分析证明解码吞吐量与编辑成本呈反比关系；
  - 实验中最高提高 15% 生成速度。
