## 摘要
大型语言模型（LLMs）在自动程序修复方面表现出高效性，但那些能通过完整测试集的合理补丁通常会修改过多代码，从而增加审查与维护成本。这种过度编辑现象十分普遍，因为大多数错误具有局部性，而标准的监督微调方法并未提供哪些 tokens 应当保留、哪些应当修改的显式信号。我们提出了 PAFT，一种面向最小编辑程序修复的保留感知微调方法。PAFT 通过对比有缺陷代码与修复后代码生成 token 级保留信号，将其与全序列掩码结合，并采用编辑难度课程训练。在 Defects4J 和 HumanEval-Java 数据集上，PAFT 相较于标准监督微调（StdFT），使 pass@1 提升最高达 65.6%，同时平均编辑距离（AED）降低最高达 32.6%。在使用 DeepSeek-Coder-6.7B 的 Defects4J 上，PAFT 还优于强大的基于偏好的修复基线 AdaPatcher，使 pass@1 从 5.9% 提升至 10.1%，并将中位 AED 从 61.0 降至 42.0。总体而言，PAFT 能够保留稳定上下文，将编辑集中于错误区域，生成更小、定位更准确的合理补丁，无需在推理阶段进行搜索、重排序或后处理。

## Abstract
Large language models (LLMs) are effective for automated program repair, but plausible patches that pass the full test suite often rewrite more code than necessary, increasing review and maintenance costs. This over-editing is common because most bugs are localized, while standard supervised fine-tuning provides no explicit signal about which tokens should be preserved and which should be changed. We propose PAFT, a preservation-aware fine-tuning method for minimal-edit program repair. PAFT derives token-level preservation signals by aligning buggy and fixed code, combines them with full-sequence masking, and applies an edit-difficulty curriculum. Across Defects4J and HumanEval-Java, PAFT improves pass@1 by up to 65.6% over standard supervised fine-tuning (StdFT) while reducing average edit distance (AED) by up to 32.6%. On Defects4J with DeepSeek-Coder-6.7B, PAFT also outperforms AdaPatcher, a strong preference-based repair baseline, improving pass@1 from 5.9% to 10.1% while reducing median AED from 61.0 to 42.0. Overall, PAFT preserves stable context and concentrates edits on faulty regions, yielding smaller, more localized, plausible patches without inference-time search, reranking, or post-processing.

## 一、核心问题与研究背景

- **研究问题**：当前大语言模型（LLMs）虽能自动生成可通过测试的修复补丁，但往往修改过多代码，即“过度编辑”（over-editing）。这导致补丁冗长、上下文破坏严重，增加了人工审查和维护成本。
- **动机与意义**：
  - 大多数程序错误具有局部性，仅需对少数代码行或词进行修改。
  - 标准监督微调（Sft）在训练中并未区分哪些 token 应保留、哪些应修改，使 LLM 缺乏关于“编辑最小化”的训练信号。
  - 实证研究表明，现有 LLM 修复补丁的规模远超人类参考补丁，暴露出系统性过度编辑问题。
- **目标**：提出一种能显式引导模型“保留稳定上下文、集中修改错误区”的训练方法，使模型生成的补丁更小、更局部、更可维护。

## 二、方法论与技术细节

### 1. 核心思想
提出 **PAFT（Preservation-Aware Fine-Tuning）**，即在微调过程中引入 token 级“保留信号”，让模型学习复制稳定代码区域，并专注修复故障区域。

### 2. 关键技术组件
- **Token 对齐与保留信号提取**  
  - 对 buggy/fixed 代码进行 token 级匹配（使用 Ratcliff/Obershelp 算法）。
  - 得到匹配块 `(i, j, n)` 后，定义 `I_align` 表示在修复代码中可直接复制的 token 位置集合。
  - 对齐 token 被视为“保留区域”，用于加权训练。

- **保留感知加权损失**  
  - 对于每个 token 位置 t，设权重：
    ```
    w_t = w_align (if t ∈ I_align), else 1
    ```
    其中 `w_align > 1`。
  - 最终损失函数：
    \[
    L(θ) = \frac{\sum M_t w_t (-\log p_θ(x_t | x_{<t}))}{\sum M_t w_t}
    \]
    - `M_t`: 掩码（assistant-only 或 full-sequence）
    - 通过对齐 token 的加权使模型倾向于复制稳定区。

- **全序列掩码 (Full Sequence Masking)**  
  - 不仅对输出代码，还在输入部分（prompt + buggy context）保留梯度信号，以维持整体上下文一致性。

- **编辑难度课程学习 (Edit-Difficulty Curriculum)**  
  - 按编辑规模（增删行数总计）排序训练样本，从“小修改”到“大修改”逐步训练。
  - 使模型先学习局部修复，再逐渐适应更复杂编辑。

- **训练架构与工具**  
  - 使用 QLoRA 对 4-bit NF4 量化的模型进行高效微调，仅训练低秩 LoRA 参数，冻结主干权重。

### 3. 推理阶段
- 推理阶段与训练模板相同，不使用对齐或加权机制。
- 无搜索、重新排序或后处理；仅一次标准自回归生成。

## 三、实验设计

### 1. 数据与评测基准
- **Defects4J**：项目级 Java 修复基准，有真实构建与测试场景。  
- **HumanEval-Java**：函数级修复任务，有单函数单元测试。
- **训练数据**：TutorLLMCode（1535对 buggy/fixed 程序），来源于 MORepair 项目。
  - 无重复样本（SHA-256 校验确认无训练-测试泄漏）。

### 2. 对比方法
- **Base**：未微调的原始模型。  
- **Sft**：标准监督微调，无保留加权。  
- **Prompting**：通过提示词“with minimal change”引导最小修改。  
- **RepairLLaMA**：高效表征与适配的程序修复模型。  
- **AdaPatcher**：基于偏好学习 (DPO) 的最小编辑修复模型。  
- **PAFT**：本文提出方法。

### 3. 评价指标
- **功能正确性**：`pass@k (k=1,5,10)` — 通过编译与测试的比例。
- **最小编辑度量**：
  - AED（Average Edit Distance）— 字符级编辑距离。
  - CCR（Code Consistency Rate）— 稳定上下文保留比。
  - 以及 ATCL / ATCT（平均变更行数/词数）。
