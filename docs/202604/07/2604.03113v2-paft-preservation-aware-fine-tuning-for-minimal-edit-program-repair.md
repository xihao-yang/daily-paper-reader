## 摘要
大型语言模型（LLMs）在自动化程序修复中表现出色，但那些通过完整测试集的合理补丁往往修改了过多代码，从而增加了代码审查与维护成本。这种过度编辑的现象很常见，因为大多数缺陷具有局部性，而标准的监督式微调并未提供明确信号来指示应保留或修改哪些标记。我们提出了 PAFT，这是一种面向保留意识的微调方法，用于最小编辑的程序修复。PAFT 通过对齐有缺陷代码与修复后代码来提取标记级别的保留信号，将其与全序列掩码相结合，并引入编辑难度课程（curriculum）。在 Defects4J 和 HumanEval-Java 基准上，PAFT 相较于标准监督微调（StdFT），pass@1 指标最高提升 65.6%，同时平均编辑距离（AED）最高减少 32.6%。在基于 DeepSeek-Coder-6.7B 的 Defects4J 数据集上，PAFT 还优于强基线方法 AdaPatcher（基于偏好学习的修复方法），在 pass@1 上从 5.9% 提升至 10.1%，同时将中位编辑距离从 61.0 降至 42.0。总体而言，PAFT 能够保留稳定的上下文，将编辑集中于有缺陷区域，生成更小、更局部化且更合理的补丁，而无需在推理阶段进行搜索、重排序或后处理。

## Abstract
Large language models (LLMs) are effective for automated program repair, but plausible patches that pass the full test suite often rewrite more code than necessary, increasing review and maintenance costs. This over-editing is common because most bugs are localized, while standard supervised fine-tuning provides no explicit signal about which tokens should be preserved and which should be changed. We propose PAFT, a preservation-aware fine-tuning method for minimal-edit program repair. PAFT derives token-level preservation signals by aligning buggy and fixed code, combines them with full-sequence masking, and applies an edit-difficulty curriculum. Across Defects4J and HumanEval-Java, PAFT improves pass@1 by up to 65.6% over standard supervised fine-tuning (StdFT) while reducing average edit distance (AED) by up to 32.6%. On Defects4J with DeepSeek-Coder-6.7B, PAFT also outperforms AdaPatcher, a strong preference-based repair baseline, improving pass@1 from 5.9% to 10.1% while reducing median AED from 61.0 to 42.0. Overall, PAFT preserves stable context and concentrates edits on faulty regions, yielding smaller, more localized, plausible patches without inference-time search, reranking, or post-processing.

## 一、研究动机与背景

- **核心问题**：大型语言模型（LLMs）在自动化程序修复（APR）中已能生成能通过测试的补丁，但这些补丁往往**修改过多代码**，即存在「过度编辑」(over-editing)。
- **实际困境**：多数代码缺陷具有地方性，只需局部修改即可修复。然而，标准的**监督式微调 (SFT)** 并未告知模型哪些部分应该保留不动、哪些需要修改，从而导致不必要的代码重写。
- **维护视角**：过度修改不仅增加审查时间，也降低代码可维护性与可追溯性。研究目标是使模型生成**更小、更集中于错误区域的补丁**。
- **研究动机**：让模型在保持测试通过能力的同时学习「不改动可稳定上下文、仅修改错误位置」的策略，实现**程序修复的可维护性优化**。

## 二、方法论：PAFT 的设计与核心技术

### 1. 核心思想
PAFT（Preservation-Aware Fine-Tuning）通过引入**保留意识（Preservation Awareness）**的训练信号，使模型在微调过程中学习“默认复制稳定部分，集中编辑错误区域”的行为。

### 2. 关键技术组成

- **（1）保留信号生成**  
  - 对每个训练样本的“错误代码 (buggy code)”与“修复代码 (fixed code)”进行**token级对齐**。  
  - 使用 **Ratcliff/Obershelp 模式匹配算法**（类似 Python 的 `SequenceMatcher`）识别匹配的连续代码块。  
  - 将匹配的 token 集合视为“应保留的部分”，并在训练时给予更高权重（权重参数 \(w_{align} > 1\)）。

- **（2）加权损失函数**
  - 在标准负对数似然损失上引入加权项：  
    \[
    L(\theta; x, M) = \frac{\sum_t M_t w_t \, (-\log p_\theta(x_t|x_{<t}))}{\sum_t M_t w_t}
    \]
  - 其中 \( w_t = w_{align} \) 若 token 属于保留区域，否则为 1。

- **（3）全序列掩蔽（Full-sequence masking）**  
  与标准“仅对回答部分计算损失”的微调不同，PAFT在整个输入输出序列上计算损失，使模型更全面理解上下文稳定性。

- **（4）编辑难度课程学习（Edit-difficulty curriculum）**  
  - 按编辑规模（新增+删除代码行数）排序样本，使模型从简单局部修改逐步学习到复杂大改动。
  - 避免早期过度复杂样本导致训练不稳定。

### 3. 微调实现细节
- 使用 **QLoRA（Low-Rank Adapter for Quantized LLMs）** 框架进行训练：  
  - 4-bit NF4 量化后冻结主模型，训练低秩适配器；既减小算力要求，又保持模型能力。
  - 学习率 2×10⁻⁴、LoRA rank=32、α=16、dropout 0.05。
- 推理阶段无额外对齐或搜索，仅标准自回归生成。

## 三、实验设计

### 1. 数据集与场景
- **Defects4J**：项目级 Java 缺陷修复基准，包含真实仓库代码与测试套件。
- **HumanEval-Java**：函数级别任务，类似代码生成场景，适用于更局部的错误修复。
- 两者共同覆盖**函数级与项目级修复场景**。

### 2. 模型与对比方法

使用 5 种模型：
- 开源骨干：Qwen3-8B、OpenCoder-8B-Instruct、DeepSeek-Coder-6.7B。
- 闭源参考：Qwen3-Max、DeepSeek-V3。

主要对比方法：
- **Base**：直接使用原始 LLM，不做微调。  
- **Sft**：标准监督微调。  
- **Prompting**：仅在指令中加入“最小改动”强调词。  
- **RepairLLaMA**：高效表示与微调的修复模型。  
- **AdaPatcher**：带偏好学习（DPO式）的强基线方法。

### 3. 评估指标
- 功能正确性：`pass@k (k=1,5,10)`。
- 编辑规模：
  - **AED (Average Edit Distance)**：字符级编辑距离（越小越好）。
  - **CCR (Code Consistency Rate)**：保留原行比例（越高越好）。
  - **ATCL / ATCT**：平均修改行数与token数。
