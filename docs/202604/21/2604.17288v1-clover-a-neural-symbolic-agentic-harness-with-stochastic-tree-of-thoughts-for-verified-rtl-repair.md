## 摘要
RTL 程序修复仍然是硬件设计与验证中的关键瓶颈。传统的自动程序修复（APR）方法依赖于预定义模板和综合手段，限制了其能够覆盖的错误类型。基于大型语言模型（LLMs）及其衍生编码代理的方案具有灵活性，但在处理较长的 RTL 代码和波形时容易出现随机性及上下文污染问题。我们提出了 Clover，一种神经—符号代理式框架，将 RTL 修复过程组织为一种针对代码操作的结构化搜索，以探索经过验证的错误修复方案。鉴于不同的修复操作对应着不同的策略，Clover 能够动态地将任务分派给专门的 LLM 代理或符号求解器。在其核心机制中，Clover 引入了随机 Tree-of-Thoughts（思想树）——一种在测试阶段的扩展机制，用搜索树的形式管理主代理的上下文，在探索与利用之间实现平衡，以确保结果的可靠性。一个专门针对 RTL 的工具箱进一步增强了代理与调试环境的交互能力。在 RTL 修复基准测试上的评估表明，Clover 在限定时间内修复了 96.8% 的错误，相较纯传统和基于 LLM 的基线方案，分别多覆盖 94% 和 63% 的错误，同时实现平均 pass@1 率为 87.5%，展示出极高的可靠性与有效性。

## Abstract
RTL program repair remains a critical bottleneck in hardware design and verification. Traditional automatic program repair (APR) methods rely on predefined templates and synthesis, limiting their bug coverage. Large language models (LLMs) and coding agents based on them offer flexibility but suffer from randomness and context corruption when handling long RTL code and waveforms. We present Clover, a neural-symbolic agentic harness that orchestrates RTL repair as a structured search over code manipulations to explore a validated solution for the bug. Recognizing that different repair operations favor distinct strategies, Clover dynamically dispatches tasks to specialized LLM agents or symbolic solvers. At its core, Clover introduces stochastic tree-of-thoughts, a test-time scaling mechanism that manages the main agent's context as a search tree, balancing exploration and exploitation for reliable outcomes. An RTL-specific toolbox further empowers agents to interact with the debugging environment. Evaluated on the RTL-repair benchmark, Clover fixes 96.8% of bugs within a fixed time limit, covering 94% and 63% more bugs than both pure traditional and LLM-based baselines, respectively, while achieving an average pass@1 rate of 87.5%, demonstrating high reliability and effectiveness.

## 二、方法论与技术框架

### 1. 整体概念

Clover整体上是一种 **多智能体（multi-agent）神经—符号框架**，它将 RTL 修复建模为一种结构化搜索过程，利用思维树（Tree-of-Thoughts，简称ToT）来引导主智能体进行多步推理与验证。

### 2. 系统架构

框架分为若干核心组件：

* **主代理（Main Agent）**：负责总体控制与修复逻辑的推演，包括提出修复假设、验证补丁、调用子代理或符号求解器。
* **上下文代理（Context Agent）**：负责在庞大的 RTL 代码中提取关键上下文片段，维护干净的 LLM 对话状态。
* **Lint-Fix 代理**：专门处理由 linter 抛出的语法或风格类错误，隔离低层噪声。
* **SMT 修复模块**：在预定义模板下调用符号求解器（扩展自 RTL-repair [11]），实现符号级逻辑修复。
* **RTL 工具箱**：内含 Verilator、Yosys、slang-server、VCD viewer 等工具，支持代码导航、模拟与波形比较。

### 3. 核心创新：随机 Tree-of-Thoughts（Stochastic ToT）

* **原理**：将 LLM 的推理轨迹组织成一个“假设树”。每个节点是一种修复思路，节点间通过验证循环派生新的修复方向。
* **改进点**：相比原始 ToT 算法的盲搜索，Clover加入**启发式采样函数**，通过一组可调权重 λ 控制探索—利用平衡。
* **启发式函数**（文字描述）：
  
  - 函数 f(代码状态 c, 历史 h) 评估节点成功修复的潜力；
  - 综合以下特征：已通过测试数、查询数量、编译错误数、token消耗、补丁数；
  - 用 softmax 分布选择下一个扩展的节点，实现随机但可控的搜索。

### 4. SMT 扩展模板

包含三类基于 RTL-Repair 的旧模板（替换字面量、添加守护、条件覆盖）及一类新增模板 **Cycle Shift**：

* Cycle Shift 模型跨周期信号延迟，利用多路选择器变量 φ 控制是否跨周期更新；
* 主代理选择合适信号，再由 SMT 求解器生成符合波形断言的修复方案；
* 修复结果不是直接修改源代码，而是返回结构化修复动作，由主代理再合成最终补丁。

## 三、实验设计与评估方案

### 1. 数据集与验证场景

* **主测试集**：RTL-Repair [11] 基准集，涵盖32个常见硬件模块（如 decoder、counter、fsm、sha3 等），其中前17个单文件模块为简单案例，后15个为复杂多文件系统。
* **对比基线**：
  - 传统符号法：RTL-Repair
  - LLM 方法：MEIC [24]、UVLLM [9]
* **评价指标**：
  - **pass@1**：单次修复成功率（波形匹配黄金参考）
  - **覆盖率**：固定时间与资源内成功修复的错误比例
  - **时间、token 使用量**：衡量效率

### 2. 消融实验（Ablation）

共分三种配置：

| 设定 | 含义 | 目的 |
|------|------|------|
| no ToT | 禁用随机思维树 | 测试 ToT 对探索能力的影响 |
| no SMT | 禁用符号修复模块，仅用 LLM | 验证神经—符号集成的增益 |
| full | 同时启用 SMT 和 ToT | 完整的 Clover 系统 |

结果表明，**no ToT** 的表现最低，**full 模式**取得最高修复率，显示两者均显著提升可靠性。

### 3. 合成数据实验（Synthetic Benchmark）

为验证泛化能力，作者随机生成无语义逻辑电路并注入八种不同错误，如操作符错误、条件反转、周期偏移等。结果显示 Clover（禁用 SMT）仍可修复多数逻辑错误，体现对新设计的适应性。
