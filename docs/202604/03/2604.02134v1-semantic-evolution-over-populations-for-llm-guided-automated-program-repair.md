## 摘要
大型语言模型（LLMs）近年来在自动程序修复（APR）中展现出强大的潜力，特别是在通过迭代优化生成和改进候选补丁的过程中。然而，最先进的基于LLM的迭代优化APR方法仍难以充分应对多项挑战，包括保持修复假设间的有用多样性、识别语义相关的修复族、组合互补的部分修复、利用结构化失败信息以及跳出结构上有缺陷的搜索区域。本文提出了一种用于APR迭代优化的人群语义进化框架，称为EvolRepair，它将基于LLM的APR形式化为一种语义进化算法。EvolRepair重新定义了经典遗传算法在APR中的搜索范式，但用由LLM及结构化执行反馈支持的语义感知组件替代了传统的基于语法的操作符。候选修复被组织为行为一致的群组，从而使算法能够保持多样性、推理修复族关系，并通过在群体中重组互补的修复见解来综合出更优的候选方案。通过利用结构化失败模式来引导搜索方向，EvolRepair既能优化有潜力的修复策略，也能在必要时转向替代抽象。实验结果表明，EvolRepair在修复效果上相较现有的基于LLM的APR方法有显著提升。

## Abstract
Large language models (LLMs) have recently shown strong potential for automated program repair (APR), particularly through iterative refinement that generates and improves candidate patches. However, state-of-the-art iterative refinement LLM-based APR approaches cannot fully address challenges, including maintaining useful diversity among repair hypotheses, identifying semantically related repair families, composing complementary partial fixes, exploiting structured failure information, and escaping structurally flawed search regions. In this paper, we propose a Population-Based Semantic Evolution framework for APR iterative refinement, called EvolRepair, that formulates LLM-based APR as a semantic evolutionary algorithm. EvolRepair reformulates the search paradigm of classic genetic algorithm for APR, but replaces its syntax-based operators with semantics-aware components powered by LLMs and structured execution feedback. Candidate repairs are organized into behaviorally coherent groups, enabling the algorithm to preserve diversity, reason over repair families, and synthesize stronger candidates by recombining complementary repair insights across the population. By leveraging structured failure patterns to guide search direction, EvolRepair can both refine promising repair strategies and shift toward alternative abstractions when necessary. Our experiments show that EvolRepair substantially improves repair effectiveness over existing LLM-based APR approaches.

## 一、核心问题与研究背景

- **研究背景**：  
  自动程序修复（Automated Program Repair, APR）旨在自动生成程序补丁，使其恢复正确行为。传统APR方法多基于语法编辑或模板匹配，容易出现过拟合、搜索空间局限和语义表达能力不足的问题。  
  随着大型语言模型（LLMs）的崛起，研究者开始使用LLM生成和迭代优化代码补丁。然而，现有LLM迭代修复方法（如 REx、ChatRepair）仍存在：
  - 修复方向缺乏多样性；
  - 语义相近但结构不同的修复族难以识别；
  - 不同候选补丁的部分正确修复无法有效整合；
  - 测试反馈利用不充分，仅以通过率为单一信号；
  - 难以跳出错误抽象导致的局部最优陷阱。

- **研究动机**：  
  作者希望通过强化语义层面的“进化式搜索”，构建一个能在群体层次上保持多样性、组合多种修复思路、并利用结构化反馈引导搜索的LLM修复框架。

## 二、方法论：EvolRepair 框架

### 1. 核心思想

- 将 **自动程序修复** 的搜索过程形式化为一种 **语义进化算法（Semantic Evolutionary Algorithm）**；
- 以 LLM 为 **语义变异算子**（Semantic Mutation Operator），生成基于代码上下文与失败反馈的修复；
- 在群体层级上维护多候选补丁，以保留和重组不同修复策略；
- 通过 **行为聚类** 和 **群体重组** 实现语义层次的多样性维护与补丁组合。

### 2. 关键技术机制

- **语义群体搜索**：候选补丁按执行行为（通过哪些测试样例）进行分组，而不是按语法或相似度分组。
- **行为聚类算法**：使用 Jaccard 相似度衡量不同补丁的测试通过集重叠度，通过层次聚类形成语义修复家族。
  
  关键公式：
  \[
  sim(P_i, P_j) = \frac{|\psi(P_i) \cap \psi(P_j)|}{|\psi(P_i) \cup \psi(P_j)|}
  \]
  其中 \(\psi(P)\) 表示补丁通过的测试案例集合。

- **语义交叉（Recombination）**：  
  - 不再限定为两个“父候选”的交叉，而是让 LLM 接收整个行为相关候选群组，整合多种修复逻辑生成更强的新补丁。
  - 使用结构化 prompt 模板指定任务约束（保持接口一致、结合强逻辑修正弱逻辑），以自然语言驱动语义融合。
  
- **语义变异（Mutation）**：  
  - 基于测试失败报告（输入、输出、错误类型）进行针对性修正；
  - 通过 LLM 生成改进版本，形成局部探索机制。

- **群体选择机制**：  
  按测试通过率（fitness）、通过测试数量及代码长度多层排序，保留 TopN 候选，维持搜索效率。

- **算法流程**（简述）：
  1. 初始化：LLM生成初始候选群体；
  2. 行为聚类：按测试通过模式划分群组；
  3. 交叉与变异：基于群组和失败反馈生成新补丁；
  4. 评估与选择：执行测试、更新群体；
  5. 达到最佳补丁或生成预算后结束。
