## 摘要
RTL 程序修复仍然是硬件设计与验证中的关键瓶颈。传统的自动程序修复（APR）方法依赖预定义模板和综合技术，这限制了其错误覆盖范围。基于大语言模型（LLM）及其编码代理的方法具有更高的灵活性，但在处理长 RTL 代码和波形时常受到随机性和上下文破坏的影响。我们提出了 Clover，这是一种神经符号智能框架，将 RTL 修复组织为对代码操作的结构化搜索，从而探索并验证针对错误的解决方案。鉴于不同的修复操作适合不同的策略，Clover 能够动态地将任务分配给专门的 LLM 代理或符号求解器。在其核心部分，Clover 引入了随机思维树机制，这是一种测试阶段的尺度扩展方法，通过将主代理的上下文管理为搜索树，在探索与利用之间实现平衡，确保结果可靠。一个专门针对 RTL 的工具箱进一步增强了代理与调试环境间的交互能力。在 RTL 修复基准上评估结果显示，Clover 在固定时间限制内修复了 96.8% 的错误，比传统方法和纯 LLM 基线分别多覆盖 94% 和 63% 的错误，同时实现了 87.5% 的平均 pass@1 率，展现出高度的可靠性与有效性。

## Abstract
RTL program repair remains a critical bottleneck in hardware design and verification. Traditional automatic program repair (APR) methods rely on predefined templates and synthesis, limiting their bug coverage. Large language models (LLMs) and coding agents based on them offer flexibility but suffer from randomness and context corruption when handling long RTL code and waveforms. We present Clover, a neural-symbolic agentic harness that orchestrates RTL repair as a structured search over code manipulations to explore a validated solution for the bug. Recognizing that different repair operations favor distinct strategies, Clover dynamically dispatches tasks to specialized LLM agents or symbolic solvers. At its core, Clover introduces stochastic tree-of-thoughts, a test-time scaling mechanism that manages the main agent's context as a search tree, balancing exploration and exploitation for reliable outcomes. An RTL-specific toolbox further empowers agents to interact with the debugging environment. Evaluated on the RTL-repair benchmark, Clover fixes 96.8% of bugs within a fixed time limit, covering 94% and 63% more bugs than both pure traditional and LLM-based baselines, respectively, while achieving an average pass@1 rate of 87.5%, demonstrating high reliability and effectiveness.

## 二、方法论与核心技术

### 1. 框架总体架构
Clover 被设计为一个**多智能体（multi-agent）神经符号系统**，可协调多种修复策略。
- **主代理（main agent）**：负责总体调度和修复流程。
- **子代理**：
  - **上下文代理（context agent）**：抽取关键代码片段，保持干净的 LLM 上下文。
  - **Lint 修复代理（lint-fix agent）**：处理由 linter 检测出的语法与风格错误。
- **符号修复模块（SMT-based solver）**：精准处理布尔表达式、字级逻辑、时序错误等低层次问题。

### 2. 神经与符号协同机制
- 主代理在每一轮修复循环中执行：
  1. **假设生成（hypothesis loop）**；
  2. **验证循环（validation loop）**；
  3. **修补循环（patch loop）**。
- 调整策略为：根据错误类型动态决定由 LLM 或 SMT 执行子任务，实现**多层次异质修复（multi-step heterogeneity）**。

### 3. 随机思维树（Stochastic Tree-of-Thoughts）
为解决 LLM 的随机性问题，作者设计了一种**测试期扩展（test-time scaling）算法**。  
其核心思想为：
- 将代理的推理过程组织为**假设树（hypothesis tree）**；
- 每个节点表示一个可能的修复假设；通过**启发式函数**评估每个节点的“潜力”；
- 利用**概率采样**在“探索”（exploration）与“利用”（exploitation）之间取得平衡。

#### 启发式函数（公式摘要）
定义节点状态 \( s = (c, h) \)，其中 \( c \) 是代码状态，\( h \) 是对话历史。  
函数形式：
\[
f(c,h) = \lambda_1 \frac{tb_p(c)}{N_{tb}} + \lambda_2 N_Q(h) - \lambda_3 N_{CE}(h) - \lambda_4 N_{tok}(h) - \lambda_5 N_P(h) + b
\]
- 每项含义：通过测试数量、查询频次、编译错误、令牌使用量与修补次数等指标综合评估节点可靠性。  
- 节点选择概率：
\[
Pr[s] = \frac{e^{f(c,h)}}{\sum_i e^{f(c_i, h_i)}}
\]

### 4. 符号修复模块的模板扩展
在现有 RTL-Repair 基础上新增模板：
- **周期偏移（Cycle Shift）模板**：通过动态选择信号是否跨周期寄存实现时序错误修复。
- 主代理选择模板、调用 SMT 形成约束问题，得到结构化修复结果，再由 LLM 合成最终源代码补丁。
