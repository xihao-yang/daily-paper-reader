## 摘要
软件安全依赖于有效的漏洞检测与修补，然而确定一个补丁是否彻底消除风险仍是一个尚未充分研究的挑战。现有的漏洞基准往往将已打补丁的函数视为本质上安全，忽视了残余安全风险的可能性。在本研究中，我们分析了 PrimeVul 基准数据集中的漏洞—良性函数对，采用多种代码语言模型（Code LMs）捕捉语义相似性，并结合基于 Tree-sitter 的抽象语法树（AST）分析来衡量结构相似性。在这些信号的基础上，我们提出了 Residual Risk Scoring（RRS），一个统一框架，整合了基于嵌入的语义相似性、局部化的 AST 结构相似性以及跨模型一致性，用于估算代码中的残余风险。我们的分析表明，良性函数在语义和结构上往往与其易受攻击的对应函数高度相似，暗示残余风险可能持续存在。进一步研究发现，约有 61% 的高 RRS 代码对呈现出 13 种不同类别的残余问题（如空指针解引用、不安全的内存分配），这些问题通过包括 Cppcheck、Clang-Tidy 和 Facebook-Infer 在内的先进静态分析工具得到了验证。结果表明，代码层面的相似性为补丁后的检查优先级提供了有效信号，使得在实际开源软件流水线中实现更可靠且可扩展的安全评估成为可能。

## Abstract
Software security relies on effective vulnerability detection and patching, yet determining whether a patch fully eliminates risk remains an underexplored challenge. Existing vulnerability benchmarks often treat patched functions as inherently benign, overlooking the possibility of residual security risks. In this work, we analyze vulnerable-benign function pairs from the PrimeVul, a benchmark dataset using multiple code language models (Code LMs) to capture semantic similarity, complemented by Tree-sitter-based abstract syntax tree (AST) analysis for structural similarity. Building on these signals, we propose Residual Risk Scoring (RRS), a unified framework that integrates embedding-based semantic similarity, localized AST-based structural similarity, and cross-model agreement to estimate residual risk in code. Our analysis shows that benign functions often remain highly similar to their vulnerable counterparts both semantically and structurally, indicating potential persistence of residual risk. We further find that approximately $61\%$ of high-RRS code pairs exhibit $13$ distinct categories of residual issues (e.g., null pointer dereferences, unsafe memory allocation), validated using state-of-the-art static analysis tools including Cppcheck, Clang-Tidy, and Facebook-Infer. These results demonstrate that code-level similarity provides a practical signal for prioritizing post-patch inspection, enabling more reliable and scalable security assessment in real-world open-source software pipelines.

- **核心问题**：传统的软件安全研究在漏洞检测和补丁生成方面取得很大进展，但往往**假定补丁后的代码完全安全**。作者指出，这一假设忽视了“残余风险”（Residual Risk）——补丁之后依旧可能存在潜在漏洞的情况。

## 二、方法论与核心思想

- **总体框架：Residual Risk Scoring (RRS)**  
  将残余风险定义为“漏洞函数与其良性版本在语义和结构上仍保持高度相似”的现象。  
  RRS 综合三类信号：语义相似性、结构相似性和跨模型一致性。
  
### 1. 语义相似性（Semantic Similarity）
- 利用多种 **Code Language Models (Code LMs)**（CodeBERT、GraphCodeBERT、UniXCoder、CodeT5-base、CodeT5-770M）生成函数的嵌入表示。
- 相似度计算公式：  
  \[
  S_{\text{cos}}(f_v, f_b) = \frac{X_m \cdot Y_m}{||X_m|| \; ||Y_m||}
  \]
  高值表示补丁后的函数仍保持与漏洞版本极高的语义相似度。  
- 比较了多种距离度量（欧氏、曼哈顿、余弦等），最终选择余弦相似度因其稳定性与归一性。

### 2. 结构相似性（Structural Similarity）
- 使用 **Tree-sitter** 生成代码的抽象语法树（AST），并计算局部化的树编辑距离（Localized TED）：
  \[
  S_{\text{AST}} = 1 - \frac{TED(T_v, T_b)}{\max(|T_v|, |T_b|)}
  \]
- 作者指出传统 **全局树编辑距离（Global TED）** 会高估变化量，而局部 TED 能更好捕捉补丁相关的细微结构变动。

### 3. 跨模型一致性（Cross-model Agreement）
- 通过多个模型的平均语义相似度与方差衡量一致性：
  \[
  \bar S_{\text{sem}} = \frac{1}{M} \sum_{m=1}^{M} S_{\text{cos}}^m,
  \quad
  C_{\text{agree}} = 1 - \frac{\sigma_{\text{cross}}}{\sigma_{\max}}
  \]
  一致性高代表不同模型对该代码的语义评估趋同，风险信号更可信。

### 4. 综合评分公式
- 最终残余风险得分定义为：
  \[
  RRS(f_v, f_b) = \alpha \cdot \bar S_{\text{sem}} + \beta \cdot R_{\text{struct}} + \gamma \cdot C_{\text{agree}}
  \]
  其中，权重设为 α=0.5、β=0.3、γ=0.2，论文还进行了权重敏感性分析以验证稳定性。

## 六、主要结论与发现

  - 高度语义与结构相似性可作为“残余风险”指示信号。
