## 摘要
在代理式管线中部署的学习型分类器面临一个根本性的可靠性问题：其预测是概率推断，而非经验证的结论，如果在缺乏可观察证据支撑的情况下据此采取行动，将导致下游阶段的连锁性失败。软件漏洞分析使这种代价变得具体且可量化。我们通过一个统一的跨语言漏洞生命周期框架来解决这一问题，该框架围绕三个由大型语言模型（LLM）驱动的推理阶段构建——混合结构-语义检测、基于执行的代理验证以及验证感知的迭代修复——其受一个严格的不变条件约束：未经基于执行的可利用性确认，不进行任何修复操作。跨语言泛化能力通过“通用抽象语法树”（Universal Abstract Syntax Tree，uAST）实现，将 Java、Python 和 C++ 规范化为共享的结构模式，并结合 GraphSAGE 与 Qwen2.5-Coder-1.5B 嵌入的混合融合，通过学习的双向门控机制实现。该机制在每个样本级别的权重提供了无需额外代价的内在可解释性。该框架在单语言检测任务中达到 89.84–92.02% 的准确率，在零样本跨语言评估中 F1 分数为 74.43–80.12%，端到端修复 69.74% 的漏洞，总失败率为 12.27%。消融实验验证了各组件的必要性：移除 uAST 会使跨语言 F1 分数下降 23.42%，禁用验证则使不必要的修复增加 131.7%。这些结果表明，基于执行的闭环推理是一种具有原则性且可实际部署的机制，可用于实现可信赖的 LLM 驱动代理式人工智能。

## Abstract
Learned classifiers deployed in agentic pipelines face a fundamental reliability problem: predictions are probabilistic inferences, not verified conclusions, and acting on them without grounding in observable evidence leads to compounding failures across downstream stages. Software vulnerability analysis makes this cost concrete and measurable. We address this through a unified cross-language vulnerability lifecycle framework built around three LLM-driven reasoning stages-hybrid structural-semantic detection, execution-grounded agentic validation, and validation-aware iterative repair-governed by a strict invariant: no repair action is taken without execution-based confirmation of exploitability. Cross-language generalization is achieved via a Universal Abstract Syntax Tree (uAST) normalizing Java, Python, and C++ into a shared structural schema, combined with a hybrid fusion of GraphSAGE and Qwen2.5-Coder-1.5B embeddings through learned two-way gating, whose per-sample weights provide intrinsic explainability at no additional cost. The framework achieves 89.84-92.02% intra-language detection accuracy and 74.43-80.12% zero-shot cross-language F1, resolving 69.74% of vulnerabilities end-to-end at a 12.27% total failure rate. Ablations establish necessity: removing uAST degrades cross-language F1 by 23.42%, while disabling validation increases unnecessary repairs by 131.7%. These results demonstrate that execution-grounded closed-loop reasoning is a principled and practically deployable mechanism for trustworthy LLM-driven agentic AI.

## 一、研究背景与核心问题

- **研究动机**  
  当前的人工智能代理系统（Agentic AI）在高风险决策场景中存在可靠性问题：LLM 的预测本质上是概率性推断，而非经过验证的事实。若基于这些预测直接采取行动，会形成“错误的连锁效应”——虚假警报、无效修复以及信任丧失。  
- **应用场景**  
  软件漏洞分析被视为理想验证领域，因为漏洞的“可利用性”可以通过程序执行客观验证，提供量化反馈。  
- **核心挑战**  
  - 预测的不确定性累积导致下游阶段失效。  
  - 现有漏洞检测与修复方法多为语言隔离式（每种编程语言需独立模型），缺少跨语言迁移能力。  
  - 多阶段管线中缺乏实证验证环节，使修复过程风险高、可信度不足。

## 二、方法论与框架说明

### 1. 总体思想  
作者设计了一个 **三阶段闭环框架（Inference–Grounding–Action）**，核心原则是**“未经执行验证不进行修复”**。  
框架由三部分构成：
1. **混合结构–语义检测（Hybrid Detection）**
2. **基于执行的代理验证（Execution-Grounded Agent Validation）**
3. **验证感知的迭代修复（Validation-Aware Iterative Repair）**

### 2. 技术细节

#### （1）通用抽象语法树（uAST）
- 通过 Tree-sitter 构建一个跨语言统一的四层结构：
  - 元数据层、节点数组层、通用节点分类层（47类）、跨语言语义映射层。
- 将 Java、Python、C++ 的语法结构映射到统一语义空间，实现零样本跨语言泛化。

#### （2）混合检测模型
- 两个分支：
  - **结构分支**：GraphSAGE 编码 uAST 图结构 → 128维嵌入。
  - **语义分支**：Qwen2.5-Coder-1.5B 提取文本语义 → 投影到 128维空间。
- **双向门控融合机制**  
  每个样本都有两个权重 \( \alpha_g,\alpha_l \)，通过 softmax 归一化，决定结构和语义信息的占比：
  \[
  \hat{h}_i = \alpha_g \hat{g}_i + \alpha_l \hat{l}_i
  \]
  权重直接可解释，提供无成本的模型内解释性。  
- **损失函数**  
  组合三项：交叉熵损失 + InfoNCE 对比损失 + 拉普拉斯正则项，用于保证模态一致性与图平滑性。

#### （3）基于执行的验证代理
- 使用 Qwen2.5-Coder-1.5B-Instruct 生成漏洞利用假设（payload、环境条件等）。  
- 语言特定执行沙箱：
  - Java：AspectJ 字节码插桩 + SecurityManager
  - Python：Monkeypatch 动态修改函数接口
  - C++：AddressSanitizer、UBSan
- 依据执行结果分类为 **确认 / 暂示 / 中性** 证据，只有确认样本进入修复阶段。

#### （4）迭代修复阶段
- 仅对已验证漏洞样本进行修复。
- 使用 LoRA 微调的 Qwen2.5-Coder-1.5B-Instruct 生成最小补丁。  
- AST 层面自动应用补丁并重新检测，直到漏洞标记为零或达到迭代上限（最多5次）。  
- 对失败样本输出完整诊断轨迹以供人机协同分析。
