## 摘要
由于需要在庞大且不断演变的代码库上进行长时间推理，并在相互依赖的行动之间保持一致性，基于大型语言模型（LLM）的代理在自动化真实世界的软件工程任务方面仍然面临挑战。现有方法通常依赖静态提示策略或人工设计的启发式方法来选择诸如代码编辑、文件导航和测试执行等动作，但缺乏对中间决策的细粒度反馈。这导致了探索效率低下、错误传播和不稳定的解题轨迹。为了解决这一限制，我们提出了 SWE-Shepherd，这一框架引入了过程奖励模型（Process Reward Models, PRMs），为代码库级别的代码代理提供密集的、逐步级别的监督。利用来自 SWE-Bench 的轨迹，我们构建了一个动作级奖励数据集，并在基础 LLM 上训练了一个轻量级奖励模型，以估计中间动作的有用性。在推理阶段，PRM 评估候选动作并引导代理朝更高奖励的决策前进，而无需完整的强化学习。对 SWE-Bench Verified 的实验结果表明，该方法提高了交互效率和动作质量，同时也揭示了在将中间奖励与最终任务成功对齐方面的挑战。

## Abstract
Automating real-world software engineering tasks remains challenging for large language model (LLM)-based agents due to the need for long-horizon reasoning over large, evolving codebases and making consistent decisions across interdependent actions. Existing approaches typically rely on static prompting strategies or handcrafted heuristics to select actions such as code editing, file navigation, and test execution, but they lack fine-grained feedback on intermediate decisions. This leads to inefficient exploration, error propagation, and brittle solution trajectories. To address this limitation, we propose SWE-Shepherd, a framework that introduces Process Reward Models (PRMs) to provide dense, step-level supervision for repository-level code agents. Using trajectories from SWE-Bench, we construct an action-level reward dataset and train a lightweight reward model on a base LLM to estimate the usefulness of intermediate actions. During inference, the PRM evaluates candidate actions and guides the agent toward higher-reward decisions without requiring full reinforcement learning. Experiments on SWE-Bench Verified demonstrate improved interaction efficiency and action quality, while also highlighting challenges in aligning intermediate rewards with final task success.

  - 探索效率低；
  - 错误积累；
  - 结果不稳定。

## 二、方法论：SWE-Shepherd 框架

### 1. 核心思想
SWE-Shepherd 的目标是为代码智能体建立 **步骤级的奖励评估机制**，在不使用完整强化学习的前提下，通过 PRM 引导智能体在每步行动中做出高价值的决策。

### 2. 总体流程（六阶段）
论文提出的 SWE-Shepherd 框架包括以下六个主要阶段（对应论文 Figure 1）：

1. **任务收集（Task Collection）**  
   从 SWE-Bench 数据集获取带有真实 GitHub 问题、仓库快照和测试的任务。

2. **轨迹生成（Trajectory Collection）**  
   使用基线 LLM 代理生成多步交互轨迹，包括“动作–观察（action–observation）”序列，如文件读取、代码编辑、测试执行等。

3. **奖励计算（Reward Computation）**  
   通过启发式规则对每个中间动作分配奖励值（0–1 范围），指标包括：  
   - 成功执行率  
   - 相关文件访问  
   - 修改目标文件  
   - 测试通过率  
   - 避免重复操作  
   并采用折扣累积奖励计算长期贡献。

4. **数据构建（Dataset Construction）**  
   将（问题描述 + 操作历史 + 当前动作）映射为奖励标签，形成动作级监督样本，最终获得约 **15,000 条训练样本**。

5. **PRM 训练（Process Reward Model Training）**  
   - PRM 在基础语言模型上添加一个轻量级 MLP Head；
   - 使用 qLoRA 进行参数高效微调；
   - 以 **MSE 损失** 优化预测的奖励值；
   - 输出为“候选动作的预测奖励”。

6. **推理阶段：奖励引导搜索（Reward-Guided Inference）**  
   在生成动作时，代理评估多个候选动作的预测奖励，选择奖励分数最高的执行，实现“奖励感知搜索”。

### 3. 理论特征
- PRM 在监督学习与强化学习之间起到折中；
- 提供密集反馈而无需显式 RL 环境；
- 能提升决策可解释性与行为一致性。

## 三、实验设计

### 1. 数据与场景
- **训练集**：SWE-Bench（共 2,294 个 GitHub 问题）  
  - 其中 1,794 个任务用于训练，  
  - 500 个任务构成 SWE-Bench Verified 测试子集。  

### 2. 评测设置
- 从 SWE-Bench Verified 中抽取 100 个任务；
- 每个任务最多执行 30 个交互步骤；
- 目标：生成能通过全部测试的修复补丁。

### 3. 对比基线
- **mini-SWE-Agent**：强基线，顺序决策，无奖励机制。  
- **SWE-Search**：结合蒙特卡洛树搜索（MCTS）的搜索框架，提供搜索式优化。  
- **SWE-Shepherd（本文方法）**：mini-SWE-Agent + PRM 奖励引导策略。  

### 4. 评测指标
- 成功解决任务比例（% Resolved）；  
- 平均代价（Avg. $，计算/时间成本）；  
- 平均步数（Avg. Steps）。

### 5. 实验结果
| 方法 | % Resolved | Avg. $ | Avg. Steps |
|------|-------------|--------|-------------|
| SWE-Search | 31% | 0.274 | — |

**结果解读：**
- 相比搜索式方法，SWE-Shepherd 具有更高效率和更低计算成本；
