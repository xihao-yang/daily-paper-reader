## 摘要
代码生成在软件工程中具有重要意义，而基于可验证奖励的强化学习（Reinforcement Learning with Verifiable Rewards, RLVR）是一种通过执行反馈提升代码生成能力的强大范式。然而，大多数RLVR流程依赖人工编写的测试，使得进展受限于稀缺且昂贵的监督。现有研究尝试使用自生成测试来构建奖励机制，但由于模型在测试生成上的性能不佳导致缺乏足够区分性的测试，从而限制了效果。我们旨在通过代码与测试生成的协同进化，在无真实标签监督的情况下改进代码生成，使它们的交互产生逐步更有信息量的监督信号。为此，我们提出了ZeroCoder，这是一种完全无标签的协同进化框架，通过自生成的代码-测试交互所产生的执行反馈联合训练Coder和Tester。针对每个问题，ZeroCoder将采样的解决方案与采样的测试进行执行，形成通过矩阵，并通过一个可插拔的选择算法识别出可能正确的解决方案与一致的测试组成的共识子集，从而生成角色特定的奖励。为保证奖励质量，ZeroCoder通过基于排序的预筛选过滤低信息实例，并利用兼顾有效性与突变驱动区分性的课程训练Tester。我们进一步识别到选择器漂移（selector drift）问题，即在协同进化过程中固定选择规则的逐步失准，并提出DyB4，这是一种贝叶斯选择器，仅需10个有标签实例即可动态重校其先验。基于三个模型和六个基准测试，ZeroCoder在代码生成与测试生成方面均表现出稳定提升。在完全无标签条件下，ZeroCoder在Qwen2.5-Coder-7B-Instruct上较基础模型的代码生成性能提升最高可达14.5%；结合DyB4后，提升幅度达到21.6%，而测试生成性能提升24.3%，接近基于监督的理想表现。

## Abstract
Code generation is important in software engineering, and Reinforcement Learning with Verifiable Rewards (RLVR) is a powerful paradigm to improve it through execution-based feedback. However, most RLVR pipelines rely on human-curated tests, making progress bottlenecked by scarce and costly supervision. Existing work tried to use self-generated tests to ground rewards, but the lack of discriminative tests constrains the effect due to the sub-optimal performance of the model on test generation. We aim to improve code generation without ground-truth supervision by co-evolving code and test generation, so that their interactions yield progressively more informative supervision. To this end, we present ZeroCoder, a fully label-free co-evolutionary framework that jointly trains a Coder and a Tester using execution feedback from self-generated code-test interactions. For each problem, ZeroCoder executes sampled solutions against sampled tests to form a passing matrix, identifies a consensus subset of likely-correct solutions and consistent tests via a pluggable selection algorithm, and derives role-specific rewards. To ensure reward quality, ZeroCoder filters low-information instances via rank-based pre-filtering and trains the Tester with a curriculum balancing validity and mutation-driven discriminativeness. We further identify selector drift, the progressive miscalibration of fixed selection rules during co-evolution, and introduce DyB4, a Bayesian selector that uses as few as 10 labeled instances to recalibrate its priors dynamically. Across three models and six benchmarks, ZeroCoder consistently improves code generation and test generation. In the fully label-free setting, it improves code generation by up to 14.5% over the base model on Qwen2.5-Coder-7B-Instruct. With DyB4, the gain reaches 21.6%, while test generation improves by 24.3%, approaching oracle-supervised performance.

## 一、研究动机与核心问题

- **背景**：  
  当前代码生成的强化学习范式主要依赖“可验证奖励”（Reinforcement Learning with Verifiable Rewards, RLVR），即通过执行测试反馈来优化模型性能。

- **核心问题**：  
  能否**在没有任何人工标注或真实监督**（ground-truth supervision）的情况下，仅依靠模型自生成的代码与测试，实现有效的代码生成优化？

- **研究目标**：  
  构建一个**完全无标签（label-free）训练框架**，使模型通过“自演化”的代码与测试交互学习自身改进，从而摆脱外部监督依赖，提升代码生成与测试生成质量。

## 二、方法论：ZeroCoder 框架

### 1. 核心思想

- 将 LLM 同时作为：
  - **Coder**：生成候选代码解决方案；
  - **Tester**：生成候选测试样例；
  两者通过互相执行产生反馈矩阵，构建用于强化训练的奖励信号。

- 通过**共演化（co-evolution）机制**实现自我增强：
  - 优秀的测试使 Coder 面临更严格约束，生成更健壮代码；
  - 高质量代码反过来推动 Tester 学习更具区分力的测试。

### 2. 关键技术模块

#### （1）Passing Matrix（执行矩阵）
- 对每个问题，执行所有代码–测试对，生成矩阵 \(E \in \{0,1\}^{N\times M}\)，其中 \(E_{ij}=1\) 表示代码 \(s_i\) 通过测试 \(t_j\)。

#### （2）Selector（选择算法）
- 根据 \(E\) 选出“可能正确”的代码子集 \(C_S\) 与“一致性高”的测试子集 \(C_T\)。  
- 三种选择器：
  - **MaxPass**：优先选通过次数最多的解；
  - **CodeT**：基于执行模式分组选出最一致的群体；
  - **B4**：贝叶斯选择器，利用执行结果估计最可能的正确代码；  
  - **DyB4（Dynamic B4）**：新提出动态贝叶斯校准选择器，利用极少量（约10个）有标签样例动态更新先验，避免训练过程中的漂移和失准。

#### （3）Rank-based 预筛选 (Pre-filtering)
- 在训练前根据 \(E\) 的秩（rank）衡量样本的交互多样性。  
- 秩太低代表代码/测试均无区分力，丢弃此类低信息样本，提高后续 RL 收敛质量。

#### （4）Tester 奖励函数设计
- 奖励分三部分：
  - **代理一致奖励**：测试能通过共识代码；
  - **突变区分奖励**：基于 mutation testing，鼓励测试能识别细微错误；
  - **格式有效性奖励**：惩罚不可执行的测试。  
  - 训练中使用**课程调度（curriculum）**逐步从有效性过渡到区分性。

#### （5）交替更新策略
- 避免因为两角色同步更新引起的非平稳性，每轮迭代交替优化 Coder 与 Tester。

## 六、主要结论与发现

1. **无监督条件下**，ZeroCoder 显著提升模型性能：
   - Qwen2.5‑Coder‑7B‑Instruct 上代码生成提升 **+14.5%**，测试生成提升 **+23.9%**。
2. 加入 DyB4 动态校准后，性能进一步提升：
   - 代码生成最高提升 **+21.6%**，测试生成 **+24.3%**，接近完全有监督（Oracle）表现。
3. 分析发现固定选择器易出现 **Selector Drift**（选择规则漂移），DyB4 可稳定优化。
4. 突变奖励与rank筛选均对性能至关重要，去除任何一项都会显著退化。
5. 所需有标签样例极少（10个即可），标签利用效率高。

- ✅ **突变驱动的测试奖励设计**：将 mutation testing 引入 LLM 奖励体系。
