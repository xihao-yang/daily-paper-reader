## 摘要
大型语言模型（LLMs）在程序修复任务中表现出色，但常常受到过度编辑的影响，即过度修改会覆盖原本正确的代码，从而妨碍错误定位。我们系统地量化了这一现象的影响，并提出了精准修复任务，其目标是在最大程度上重用正确代码的同时，仅修复存在错误的部分。基于这一洞见，我们提出了 PRepair 框架，用以缓解过度编辑并提升修复准确性。PRepair 包含两个组件：自损（Self-Breaking）与自修复（Self-Repairing）。自损通过受控的错误注入和极值采样（min-max sampling）生成多样化的错误程序；自修复则利用编辑感知组相对策略优化（Edit-Aware Group Relative Policy Optimization，EA-GRPO）进行模型训练，并通过编辑感知奖励函数鼓励最小但正确的编辑。实验结果表明，PRepair 在 $\mathrm{fix}_1@1$ 指标下可将修复精度提升最多 31.4%，该指标同时考虑修复的正确性与修改范围。此外，当与推测性编辑（speculative editing）结合使用时，PRepair 显著提高了解码吞吐量，展示了其在精准且实用的代码修复中的潜力。

## Abstract
Large Language Models (LLMs) achieve strong program repair performance but often suffer from over-editing, where excessive modifications overwrite correct code and hinder bug localization. We systematically quantify its impact and introduce precise repair task, which maximizes reuse of correct code while fixing only buggy parts. Building on this insight, we propose PRepair, a framework that mitigates over-editing and improves repair accuracy. PRepair has two components: Self-Breaking, which generates diverse buggy programs via controlled bug injection and min-max sampling, and Self-Repairing, which trains models with Edit-Aware Group Relative Policy Optimization (EA-GRPO) using an edit-aware reward to encourage minimal yet correct edits. Experiments show that PRepair improves repair precision by up to 31.4% under $\mathrm{fix}_1@1$, a metric that jointly considers repair correctness and extent, and significantly increases decoding throughput when combined with speculative editing, demonstrating its potential for precise and practical code repair.

## 一、研究背景与核心问题

- **研究领域**：自动程序修复（Automated Program Repair, APR），特别是基于大型语言模型（LLMs）的代码修复。
- **问题动机**：
  - 当前LLMs在程序修复上虽取得显著进展，但存在**过度编辑（over-editing）**问题——模型为保证正确性往往大幅度改动代码。
  - 过度编辑造成两大负面影响：
    1. 破坏原有正确逻辑，使得修复结果难以定位真实错误位置；
    2. 降低可维护性与开发者审查效率。
- **研究目标**：提出一种方法，使LLM能够**精准修复**（precise repair）代码，即在保证修复正确的前提下，最大限度地保留原始正确代码，仅修改确有错误的局部。

## 二、方法论：PRepair 框架与关键算法

### 1. 核心思想
通过任务重定义与奖励机制改进，让模型在修复代码时理解“修改代价”，在正确性与编辑最小化之间取得平衡。

### 2. 框架结构
PRepair 由两个阶段组成（如图3）：
1. **Self-Breaking（自破坏）**：
   - 通过受控的“错误注入”生成多样化的 buggy 程序。
   - 使用 **min–max 采样策略** 确保错误样本间的多样性，避免集中于少数 bug 模式。
   - 生成的 bug 程序组成用于训练的高质量数据集。
2. **Self-Repairing（自修复）**：
   - 使用提出的 **EA-GRPO（Edit-Aware Group Relative Policy Optimization）** 算法对模型进行强化学习调优。
   - 奖励函数综合考虑**修复正确性**与**编辑代价**（Edit Cost），旨在鼓励模型执行最小且正确的修改。

### 3. 关键技术细节
- **编辑代价度量 (Edit Cost, DEC)**  
  \[
  DEC(X, Y) = \frac{D(X, Y)}{|X|}
  \]
  其中 \( D(X, Y) \) 为Levenshtein距离（插入、删除、替换行数），\(|X|\) 为原代码行数。
- **评估指标：fixₚ@k**
  - 首次定义用于精准修复的度量指标，综合考虑功能正确性和修改大小。
  - 当修改成本不超过设定比例 \( p \) 且程序通过测试时，视为成功修复。
- **EA-GRPO的动态奖励机制**
  - 当修复结果组的平均正确率超过阈值 α 时启动编辑惩罚；
  - 奖励函数：  
    \[
    R_i = 
    \begin{cases}
      1 - T(G)·β·P_i, & \text{若修复正确}\\
      0, & \text{否则}
    \end{cases}
    \]
    其中 \(β\) 为惩罚系数，\(P_i\) 依据组内编辑代价的均值与标准差归一化处理。

### 4. 理论推导与加速机制
- **Speculative Edits（推测编辑）**：
  - 结合 *Prompt Lookup Decoding* 理论推导表明，当编辑代价 DEC 越低时，解码吞吐量 \(T\) 随之非线性上升。
  - 因此，EA-GRPO优化不仅提高修复精度，也能**显著加快推理速度（最多提升约15%）**。
