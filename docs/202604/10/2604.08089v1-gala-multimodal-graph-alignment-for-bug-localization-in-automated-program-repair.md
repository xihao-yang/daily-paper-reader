## 摘要
基于大型语言模型（LLM）的自动程序修复（APR）在文本类基准测试中展现出强大潜力，但在包含 GUI 截图的多模态场景中表现不佳。现有方法通常将图像转换为纯文本，从而丢失关键的空间关系，导致视觉观测与代码组件之间的严重脱节，使得缺陷定位退化为不精确的关键词匹配。为弥合这一差距，我们提出 GALA（Graph Alignment for Localization in APR），一个将多模态 APR 从隐式语义猜测转向显式结构推理的框架。GALA 包含四个阶段：首先构建图像 UI 图以捕捉视觉元素及其结构关系；接着通过将该 UI 图与代码仓库级结构（如文件引用）交叉对齐，进行文件级对齐以定位候选文件；然后通过分析细粒度代码依赖（如调用图）执行函数级对齐，将视觉元素精确映射到对应的代码组件；最后在已对齐的文件与函数所确定的代码上下文中执行补丁生成。通过系统地在多模态间强制语义与关系一致性，GALA 建立了高度精确的视觉到代码映射。在 SWE-bench Multimodal 基准上的评估表明，GALA 达到了当前最先进的性能，凸显了分层结构对齐的有效性。

## Abstract
Large Language Model (LLM)-based Automated Program Repair (APR) has shown strong potential on textual benchmarks, yet struggles in multimodal scenarios where bugs are reported with GUI screenshots. Existing methods typically convert images into plain text, which discards critical spatial relationships and causes a severe disconnect between visual observations and code components, leading localization to degrade into imprecise keyword matching. To bridge this gap, we propose GALA (Graph Alignment for Localization in APR), a framework that shifts multimodal APR from implicit semantic guessing to explicit structural reasoning. GALA operates in four stages: it first constructs an Image UI Graph to capture visual elements and their structural relationships; then performs file-level alignment by cross-referencing this UI graph with repository-level structures (e.g., file references) to locate candidate files; next conducts function-level alignment by reasoning over fine-grained code dependencies (e.g., call graphs) to precisely ground visual elements to corresponding code components; and finally performs patch generation within the grounded code context based on the aligned files and functions. By systematically enforcing both semantic and relational consistency across modalities, GALA establishes a highly accurate visual-to-code mapping. Evaluations on the SWE-bench Multimodal benchmark demonstrate that GALA achieves state-of-the-art performance, highlighting the effectiveness of hierarchical structural alignment.

## 一、研究背景与核心问题

- **研究动机：**  
  自动程序修复（Automated Program Repair, APR）的目标是自动识别并修复软件错误。然而，现有基于 **大型语言模型（LLM）** 的方法主要依赖文本输入（如错误报告与代码），在面对**带有 GUI 截图的多模态缺陷报告**时表现不佳。  
- **问题根源：**
  1. 将图像转化为文本会造成**视觉结构信息丢失**，无法捕捉 UI 元素间空间或交互关系。
  2. 视觉描述与代码结构之间缺乏直接联系，使得缺陷定位退化为关键词匹配式的弱关联。  
- **研究目标：**  
  本文提出一种 **结构化的跨模态对齐机制**，在视觉层面与代码层面建立显式一致性，以实现更精确的缺陷定位与修复。

## 二、方法论（GALA 框架）

### 1. 核心思想  
GALA（**Graph Alignment for Localization in APR**）将多模态 APR 问题转化为**多层次图对齐（hierarchical graph alignment）**任务：
- 在视觉侧构建 **UI 图（Image Graph）**；
- 在代码侧构建 **多层次代码图（Code Graph）**；
- 通过分层对齐实现从 **视觉元素 → 文件 → 函数 → 补丁生成** 的结构化映射。

### 2. 四阶段流程（如图 2 所示）
1. **图像图构建（Image Graph Construction）**
   - 利用视觉语言模型（VLM）从 GUI 截图中提取 UI 元素（节点）及其空间或交互关系（边）。
   - 采用类型识别（UI 页面、图表、代码截图等）、锚点选择（root objects）与局部扩展策略只保留与问题相关的局部结构。
   - 生成问题中心、稀疏但信息充分的“根化图”（rooted graph）。

2. **文件级对齐（File-level Alignment）**
   - 从代码仓库构建文件依赖图（import/reference 关系）。
   - 将图像图的结构摘要与代码仓库层级目录作为输入，让 LLM 进行**跨模态语义比对**，筛选出候选文件并通过结构关系精炼出“种子文件（seed files）”。

3. **函数级对齐（Function-level Alignment）**
   - 在种子文件中解析函数声明、调用、状态操作等形成函数级代码图。
   - 通过跨模态图对齐验证视觉节点与函数节点之间的**语义兼容与关系一致性**，输出精确的可编辑目标函数（edit targets）。

4. **基于图的补丁生成（Graph-guided Patch Generation）**
   - 利用已对齐的文件与函数限定修复范围。
   - 修复代理在本地结构化上下文中生成最小可行补丁（使用 `git diff` 导出标准化补丁文件）。
   - 该过程结合视觉症状、依赖上下文与对齐结果进行决策，避免无关重构或过度修改。

### 3. 模型基础与实现细节  
- 整个流程均由 **Qwen3.5 系列 LLM** 实现；在不同阶段分别充当视觉语言模型、文本推理模型与代码代理。
- 对齐和生成阶段均以预设“分阶段 Prompt 模板”（如 Fig.4）指导 LLM 输出结构化 JSON 格式结论，提高可复现性与可解释性。

## 六、主要结论与发现

- **性能表现：**
  - 在 SWE-Bench M 上，GALA(Qwen3.5-122B) 达到 **35.40% 修复成功率**，为当前最优。
  - 相比 SVRepair 提升约 **+1.7%**。  
  - 定位能力提升显著：文件级召回 29.22%，函数级召回 17.14%。  

- **关键发现：**
  - 性能提升主要来自 **跨模态结构对齐机制**，而非模型规模扩展。
  - 分层结构（文件+函数）可互补：粗粒度提供全局上下文，细粒度提高定位精度。
  - 跨模态语义与关系一致性是实现精准修复的关键要素。

## 七、方法与实验亮点

- **显式结构推理：**

  框架依赖规范的文件组织与命名；在结构混乱项目中可能性能下降。
