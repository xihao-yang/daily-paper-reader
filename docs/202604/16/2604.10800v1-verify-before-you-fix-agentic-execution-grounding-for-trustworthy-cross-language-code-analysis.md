## 摘要
在代理式流水线中部署的学习型分类器面临一个根本的可靠性问题：预测是概率推断而非已验证的结论，若在缺乏可观测证据支撑的情况下基于预测采取行动，将会导致下游阶段的连锁失效。软件漏洞分析使这种代价变得具体且可度量。我们通过一个统一的跨语言漏洞生命周期框架来解决这一问题，该框架围绕三个由大语言模型（LLM）驱动的推理阶段构建：混合结构–语义检测、基于执行的代理验证，以及具备验证意识的迭代修复，所有步骤均受严格不变条件约束——在未通过执行确认漏洞可被利用之前，不进行任何修复操作。跨语言泛化通过一个通用抽象语法树（Universal Abstract Syntax Tree, uAST）实现，该结构将 Java、Python 和 C++ 规范化为统一的结构化模式，并结合 GraphSAGE 与 Qwen2.5-Coder-1.5B 嵌入的混合融合，通过双向门控学习机制实现。该机制的样本级权重提供了内在可解释性，而无需额外代价。该框架实现了 89.84–92.02% 的语言内检测准确率与 74.43–80.12% 的零样本跨语言 F1 分数，能够以 12.27% 的总失败率端到端修复 69.74% 的漏洞。消融实验验证了关键组成部分的必要性：移除 uAST 会使跨语言 F1 下降 23.42%，而禁用验证则导致不必要修复增加 131.7%。这些结果表明，基于执行的闭环推理是一种有原理且可实际部署的机制，用于实现可信赖的由 LLM 驱动的代理式人工智能。

## Abstract
Learned classifiers deployed in agentic pipelines face a fundamental reliability problem: predictions are probabilistic inferences, not verified conclusions, and acting on them without grounding in observable evidence leads to compounding failures across downstream stages. Software vulnerability analysis makes this cost concrete and measurable. We address this through a unified cross-language vulnerability lifecycle framework built around three LLM-driven reasoning stages-hybrid structural-semantic detection, execution-grounded agentic validation, and validation-aware iterative repair-governed by a strict invariant: no repair action is taken without execution-based confirmation of exploitability. Cross-language generalization is achieved via a Universal Abstract Syntax Tree (uAST) normalizing Java, Python, and C++ into a shared structural schema, combined with a hybrid fusion of GraphSAGE and Qwen2.5-Coder-1.5B embeddings through learned two-way gating, whose per-sample weights provide intrinsic explainability at no additional cost. The framework achieves 89.84-92.02% intra-language detection accuracy and 74.43-80.12% zero-shot cross-language F1, resolving 69.74% of vulnerabilities end-to-end at a 12.27% total failure rate. Ablations establish necessity: removing uAST degrades cross-language F1 by 23.42%, while disabling validation increases unnecessary repairs by 131.7%. These results demonstrate that execution-grounded closed-loop reasoning is a principled and practically deployable mechanism for trustworthy LLM-driven agentic AI.

## 一、研究动机与核心问题

- **问题背景**：大语言模型（LLM）及相关学习分类器在代码分析、诊断、决策等高风险领域展现出强大潜力，但预测结果往往只是**概率推断而非经过验证的事实**。

  - 现有软件漏洞检测与修复体系通常**分割检测、验证和修复三步**，导致工具间缺乏关联，且多数系统在未确认漏洞可被利用前即开展修复。  
- **研究目标**：提出一种可信赖的、跨语言统一的、可执行验证支撑的代码漏洞分析与修复机制，使LLM的推理过程“**修复前先验证（Verify Before You Fix）**”。

## 二、方法论与技术框架

### 1. 总体思想
- 论文提出**三阶段闭环生命周期框架**（inference–grounding–action），以“执行验证”为核心原则：  
  - **检测（Detection）**：混合结构–语义模型识别潜在漏洞。  
  - **验证（Validation）**：基于沙箱执行确认漏洞是否可被真实利用。  
  - **修复（Repair）**：仅对“经过执行确认”的漏洞进行迭代式修复。  
- **关键不变条件**：在漏洞未经实际执行确认可利用性前，系统**禁止自动修复**。

### 2. 技术组件

#### （1）通用抽象语法树（Universal AST, uAST）
- 统一 Java / Python / C++ 三种语言到共享表示层，将200余种语言节点映射为47类通用节点类型。  
- 实现四层结构：元数据层、扁平节点数组、语言无关节点范畴、跨语言语义映射。  
- 提高跨语言特征迁移性。

#### （2）混合结构–语义检测模型
- 模型架构融合 **GraphSAGE**（结构特征）与 **Qwen2.5-Coder-1.5B**（语义特征）。  
- **双向门控机制（two-way gating）**：  
  每个样本获得[α_g, α_l]两个加权系数，决定结构和语义模态的相对贡献。  
- **损失函数组合**：  
  - 交叉熵损失 \(L_{CE}\)  
  - 对比学习损失 \(L_{InfoNCE}\) 促进模态一致性  
  - 拉普拉斯正则损失 \(L_{Laplacian}\) 平滑图结构  
  - 总损失：\( L = L_{CE} + λ_{nce}L_{InfoNCE} + λ_{lap}L_{Laplacian}\)

#### （3）执行落地代理验证（Execution-Grounded Validation）
- 利用 LLM 规划器生成攻击假设、负载和输入条件。  
- 自动构建语言特定测试环境（C++ 使用 Sanitizer、Java 使用 AspectJ、Python 使用 monkeypatch）并在沙箱中实际执行。  
- 根据观察结果分为**确认 / 暗示 / 无证据**三类，至少有一项确认证据才认定为真实漏洞。

#### （4）验证感知迭代修复（Validation-Aware Iterative Repair）
- 仅对确认目标进行修复，修复模型为 **LoRA 微调版 Qwen2.5-Coder-1.5B-Instruct**。  
- 针对漏洞根因及payload生成最小修补（minimal patch），每次修复后重新检测。  
- **最多迭代五次**；若仍未修复成功，则输出结构化诊断轨迹供人工验证，从而保留人机协作空间。

## 七、方法亮点

- **创新性理念**：首次形成“Verify-before-Fix”治理原则，把执行事实引入LLM推理闭环。
