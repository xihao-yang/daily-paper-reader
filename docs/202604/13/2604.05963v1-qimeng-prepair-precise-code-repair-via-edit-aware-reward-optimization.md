## 摘要
大规模语言模型（LLMs）在程序修复方面表现出色，但常常受到过度编辑的影响，即过多的修改会覆盖正确的代码、妨碍错误定位。我们系统地量化了这种影响，并引入了精确修复任务，其目标是在修复程序漏洞的同时最大化正确代码的复用。基于这一见解，我们提出了 PRepair 框架，以减轻过度编辑并提高修复准确性。PRepair 包含两个组成部分：Self-Breaking，通过受控的漏洞注入和 min-max 采样生成多样化的含错程序；以及 Self-Repairing，利用 Edit-Aware Group Relative Policy Optimization（EA-GRPO）和基于编辑感知的奖励进行模型训练，以鼓励最少且正确的编辑。实验结果显示，PRepair 在 $\mathrm{fix}_1@1$ 指标下可使修复精度提升最高达 31.4%，该指标综合考虑修复的正确性与修改的程度；此外，与推测性编辑结合时显著提升了解码吞吐量，展现出其在精确且实用的代码修复方面的潜力。

## Abstract
Large Language Models (LLMs) achieve strong program repair performance but often suffer from over-editing, where excessive modifications overwrite correct code and hinder bug localization. We systematically quantify its impact and introduce precise repair task, which maximizes reuse of correct code while fixing only buggy parts. Building on this insight, we propose PRepair, a framework that mitigates over-editing and improves repair accuracy. PRepair has two components: Self-Breaking, which generates diverse buggy programs via controlled bug injection and min-max sampling, and Self-Repairing, which trains models with Edit-Aware Group Relative Policy Optimization (EA-GRPO) using an edit-aware reward to encourage minimal yet correct edits. Experiments show that PRepair improves repair precision by up to 31.4% under $\mathrm{fix}_1@1$, a metric that jointly considers repair correctness and extent, and significantly increases decoding throughput when combined with speculative editing, demonstrating its potential for precise and practical code repair.

## 一、研究问题与背景

- **核心问题**：本文关注的是大语言模型（LLMs）在自动程序修复（Program Repair）任务中的一个关键缺陷——**过度编辑（Over-Editing）**。  
  传统的 LLM 仅以修复正确性为优化目标，会导致模型倾向于**重写整个代码段**而非准确定位并修复错误，造成以下问题：
  - 难以精确定位 bug；
  - 改动范围过大、破坏原结构；
  - 增加开发者复审负担与降低可维护性。

- **研究动机**：作者指出，真正的程序修复应在**保持正确逻辑最大化复用的同时，仅修改必要部分**，称之为“精确修复（Precise Repair）”。

- **研究目标**：设计一种训练与评估机制，使得 LLMs 在修复过程中不仅考虑正确性，还考虑**编辑距离最小化**，从而鼓励最少且正确的编辑行为。

## 二、方法论：PRepair 框架与 EA-GRPO 算法

### 1. 整体框架

PRepair 包含两个阶段：

1. **Self-Breaking（自破坏）阶段**：  
   - 模型被提示生成“带 bug”的程序样本。  
   - 通过**受控漏洞注入**与**min–max 采样策略**确保生成的错误样本多样性与代表性。  
   - 公式化地最小化样本间的最大相似度，使得错误样本尽可能多样。

2. **Self-Repairing（自修复）阶段**：  
   - 模型基于上述错误数据进行强化学习训练；
   - 利用新的算法 **EA-GRPO（Edit-Aware Group Relative Policy Optimization）** 进行策略优化；
   - 引入**编辑感知奖励（Edit-Aware Reward）**以平衡修复正确性与编辑代价。

### 2. 核心思想

- 不再仅以“是否修复正确（通过测试）”为唯一标准，而是综合考虑：
  - **正确性 Reward**；
  - **编辑代价惩罚（Edit Cost Penalty）**；
  - 当修复达到一定准确阈值后，惩罚多余修改。

### 3. 关键技术细节

- **编辑代价（Edit Cost, DEC）**：  
  基于**Levenshtein 距离**的行级编辑距离，度量修改幅度：
  \[
  DEC(X, Y) = \frac{D(X, Y)}{|X|}
  \]
  其中 \(D(X,Y)\) 为两程序间的行级编辑距离。

- **新指标 fix p@k**：  
  同时评估修复的正确性与修改范围，定义为模型在前 k 个候选中，能在编辑代价不超过容忍比例 p 的情况下通过测试用例的比例。

- **EA-GRPO 动态奖励设计**：  
  - 当生成组（group）平均准确率 ≥ 阈值 α 时，才应用编辑惩罚；
  - 对正确样本按其编辑距离的标准化差异施加惩罚项：
    \[
    R_i = 1 - T(G) \cdot β \cdot P_i
    \]
    其中 β 控制惩罚强度，\(P_i\) 为归一化后编辑成本；
  - 使模型在“必要时修正，非必要时不动”。

- **结合推测性解码（Speculative Editing）**：
  - 通过减少修改，提高输入代码中草稿 token 的接受率；
  - 推理吞吐量与 \(1 - DEC\) 成正相关，实现高效生成。
