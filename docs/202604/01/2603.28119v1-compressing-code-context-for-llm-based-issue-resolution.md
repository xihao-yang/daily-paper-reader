## 摘要
大型语言模型（LLMs）如今能够解决真实的 GitHub 问题。然而，当前方法往往对代码上下文作过度近似，从而面临两个复合性难题：处理庞大输入的高昂成本，以及上下文窗口噪声过多导致模型偏离修复信号、降低有效性。现有的压缩技术未能解决这一张力：通用压缩器会损害代码的语义完整性，而代码专用工具则缺乏对代码结构及任务上下文的感知，无法保留关键修补成分。为此，我们提出了一个包含两个组件的新框架。首先是 Oracle-guided Code Distillation（OCD），一种上下文蒸馏算法，结合遗传搜索与差分调试系统性地将代码上下文缩减至最小充分子序列——仅保留实现成功修复所需的成分。然后利用蒸馏后的数据对 SWEzze 进行微调，这是一种轻量模型，可在推理时学习压缩代码上下文，过滤噪声、减轻干扰，同时保留修复所需成分。在 SWE-bench Verified 数据集上对三种前沿 LLM 进行评估，SWEzze 在不同模型中保持约 6 倍的稳定压缩率，相对未压缩设置减少 51.8%-71.3% 的总 token 预算，使问题解决率提升 5.0%-9.2%，并在有效性、压缩比与延迟之间实现了相较于最新上下文压缩基线方法的最佳综合平衡。

## Abstract
Large Language Models (LLMs) are now capable of resolving real-world GitHub issues. However, current approaches overapproximate the code context and suffer from two compounding problems: the prohibitive cost of processing massive inputs, and low effectiveness as noise floods the context window and distracts the model from the bug-fixing signal. Existing compression techniques fail to resolve this tension: generic compressors compromise the semantic integrity of code, while code-specific tools lack awareness of code structure and task context to preserve essential patch ingredients. To address this, we propose a novel framework consisting of two components. First, Oracle-guided Code Distillation (OCD), a context distillation algorithm that combines genetic search and delta debugging to systematically reduce code contexts to their minimal sufficient subsequence - retaining only the ingredients required for a successful fix. We use this distilled data to fine-tune SWEzze, a lightweight model that learns to compress code context at inference time, filtering noise and combating distraction while preserving fix ingredients. Evaluated on SWE-bench Verified across three frontier LLMs, SWEzze maintains a stable compression rate of about 6 times across models, reduces the total token budget by 51.8%-71.3% relative to the uncompressed setting, improves issue resolution rates by 5.0%-9.2%, and delivers the best overall balance among effectiveness, compression ratio, and latency compared with state-of-the-art context compression baselines.

## 1. 核心问题与研究背景  

- **研究动机**：  
  当前大型语言模型（LLM）在处理 GitHub 真实问题（issue resolution）时，必须从庞大仓库中提取相关代码上下文。但现有方法往往采用启发式检索或 RAG（Retrieval-Augmented Generation），返回大量冗余文件或函数，造成：

  如何在不破坏代码语义和结构完整性的前提下，有效压缩上下文，仅保留真正有助于修复的“**fix ingredients**”（关键修补成分）。  

- **研究意义**：  
  通过构建一个能精确筛选必要代码片段的压缩系统，降低成本，同时提升 LLM 对真实代码问题的处理成功率。

### 2.2 Oracle-Guided Code Distillation (OCD)  
**目标**：在给定问题描述 \(I\)、错误位置 \(F\)、初始上下文 \(C_{init}\) 中，找到一个最小充分子集 \(\hat{C} \subset C_{init}\)，使得 LLM 仅用 \(\hat{C}\) 即可生成通过测试的修复补丁。  

**创新思想**：  
- 通过**功能充分性（functional sufficiency）**而非词法或相似性标准来定义压缩目标；  
- 利用执行验证（oracle）判定候选上下文是否足以生成可通过测试的补丁。  

**算法流程**（两阶段搜索）：  
1. **遗传算法 (Genetic Algorithm, GA)**  
   - 将每个代码结构单元（文件 / 函数 / 语句块）编码为基因位。  
   - 以优先级打分（Φ）初始化与进化搜索，有效探索大量上下文组合。  
   - 目标是找到一个“能修复问题”的候选上下文。  
2. **分层差分调试（Hierarchical Delta Debugging, HDD）**  
   - 在 GA 找到可行上下文后，进一步层层删减文件 → 函数 → 代码块；  
   - 直到去除任一单元都会导致修复失败，达到**1-最小充分性（1-minimality）**。  

**优先级Φ构成**：
\[
\Phi(u) = w_p \cdot I(u \in F_{patch}) + 
w_c \cdot \log(1 + |L_{cov}(u)|) + 
w_s \cdot SymScore(u, P_{gold})
\]
- 其中各部分代表：  
  - `Patch Overlap`：文件是否被人类补丁修改；
  - `Test Coverage`：该代码单元被失败测试覆盖的行数；
  - `Symbol Overlap`：与补丁符号（函数名、变量名）匹配度。  

---

### 2.3 SWEzze 模型  
**核心功能**：在推理时根据训练好的模型预测每个片段应否保留，实现在线上下文压缩。  

**技术细节**：  
- 基于 **Qwen3-Reranker-0.6B** 微调的**跨编码器模型**；  
- 使用 LoRA（Low-Rank Adaptation）做参数高效微调（rank=16, α=32）；  
- 输入为结构化查询 `(issue description + fault location)` 与代码片段；  
- 输出是否保留的概率分数 \( f(q, s; \theta) \in [0,1] \)。  
- 训练时用 OCD 蒸馏数据作为监督信号；同时采用
  - **类别权重**：增强正样本影响；
  - **角色权重（role-aware weighting）**：不同语义角色（如 schema、definition、utility）赋予不同权重；  
  - **预算感知贪心选择（budget-aware greedy selection）**控制最终token数量。
