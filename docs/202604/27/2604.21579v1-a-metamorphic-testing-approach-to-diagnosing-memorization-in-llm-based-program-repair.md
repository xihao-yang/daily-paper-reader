## 摘要
基于大型语言模型（LLM）的自动程序修复（APR）技术在降低调试成本方面表现出良好前景。然而，已有结果可能受到数据泄漏的影响：当评估基准与模型的预训练数据重叠时，LLM 可能会记忆缺陷修复，从而导致性能估计被高估。本文探讨了一种方法，即将变形测试（MT）与先前研究中作为记忆化代理指标使用的负对数似然（NLL）相结合，以更好地揭示数据泄漏问题。我们通过对两个广泛使用的数据集——Defects4J 和 GitBug-Java——应用保持语义不变的代码转换，构建了变体基准集。基于这些基准，我们评估了七个 LLM 在原始和转换版本上的修复成功率，并分析了性能下降与 NLL 之间的关系。结果显示，所有评估的先进 LLM 在转换基准上的补丁生成成功率均出现显著下降，从 GPT-4o 的 -4.1% 到 Llama-3.1 的 -15.98% 不等。此外，我们发现这种性能下降与原始基准上的 NLL 呈强相关，表明模型在更可能被其记忆的实例上表现更好。这些发现表明，将 MT 与 NLL 结合能够提供更有力且更可靠的数据泄漏证据，而单独使用变形测试也有助于缓解 LLM 基于 APR 评估中的数据泄漏效应。

## Abstract
LLM-based automated program repair (APR) techniques have shown promising results in reducing debugging costs. However, prior results can be affected by data leakage: large language models (LLMs) may memorize bug fixes when evaluation benchmarks overlap with their pretraining data, leading to inflated performance estimates. In this paper, we investigate whether we can better reveal data leakage by combining metamorphic testing (MT) with negative log-likelihood (NLL), which has been used in prior work as a proxy for memorization. We construct variant benchmarks by applying semantics-preserving transformations to two widely used datasets, Defects4J and GitBug-Java. Using these benchmarks, we evaluate the repair success rates of seven LLMs on both original and transformed versions, and analyze the relationship between performance degradation and NLL. Our results show that all evaluated state-of-the-art LLMs exhibit substantial drops in patch generation success rates on transformed benchmarks, ranging from -4.1% for GPT-4o to -15.98% for Llama-3.1. Furthermore, we find that this degradation strongly correlates with NLL on the original benchmarks, suggesting that models perform better on instances they are more likely to have memorized. These findings show that combining MT with NLL provides stronger and more reliable evidence of data leakage, while metamorphic testing alone can help mitigate its effects in LLM-based APR evaluations.

- **研究背景**  
  自动程序修复（Automated Program Repair, APR）旨在通过自动生成补丁以修复软件漏洞，降低开发人员调试成本。近年来，大型语言模型（LLM）如 ChatGPT、Claude、Llama 等在 APR 任务中表现突出。  
  然而，随着广泛使用的基准（如 Defects4J）被重复采用于多个研究中，**数据泄漏（data leakage）和记忆化（memorization）**成为关键隐患——即 LLM 在训练阶段可能已接触到相应的缺陷与修复样本，导致评估性能被虚高。

- **核心问题**

- **研究动机**
  - 数据泄漏难以彻底避免（因公开数据广泛复用且 LLM 训练语料不透明）。

## 2. 方法论：结合变形测试（MT）与负对数似然（NLL）

### 2.1 核心思想
- 通过 **变形测试（Metamorphic Testing, MT）** 对源代码进行 **语义等价但表面结构变化的转换**；
- 比较模型在原始代码与变形代码上的修复成功率差异；
- 将这种性能下降与 **负对数似然（Negative Log-Likelihood, NLL）** 指标关联分析，以此揭示模型对训练数据的熟悉度和记忆程度。

### 2.2 技术流程
整体流程分三阶段：
1. **Transform 阶段**：  
   - 使用自研框架 *CodeCocoon* 对代码进行语义保持的变换，如变量重命名、控制结构重写、条件调换等（详见下方 Table 1 类型）。
2. **Generate Patch 阶段**：  
   - 各 LLM 对原始 & 变形后的代码生成修复补丁；提示词包含函数源码、Javadoc、失败测试和栈追踪信息；采用 Chain-of-Thought 推理方式。
3. **Evaluate Patch 阶段**：  
   - 在项目测试套件上执行生成的补丁，计算 **修复成功率（SR）**：
     \[
     SR = \frac{\text{成功修复的次数}}{\text{总提示次数}}
     \]
   - 对比原始与变形版本差异（ΔSR），衡量鲁棒性。
4. **统计检验**：
   - 使用 **Wilcoxon 符号秩检验** 检验显著性；
   - 利用 **Vargha–Delaney ˆA12** 衡量效应强度；
   - Spearman 相关系数计算 ΔSR 与 NLL 的相关性。

### 2.3 关键概念
- **NLL（负对数似然）** 用于度量模型对序列的“熟悉度”，NLL 低表示模型可能在训练中见过类似模式。
- 若在低 NLL 样本上变形后性能显著下降，则可推断这些样本存在记忆化或泄漏。

## 6. 主要发现与结论

### 6.1 总体结果
- **Defects4J 上所有模型均出现显著性能下降**：
  - 平均成功率下降幅度从 GPT‑4o 的 −4.01% 到 Llama‑3.1 的 −15.98%；  
  - 均达到统计显著（p < 0.001）。  
- **GitBug‑Java 上下降幅度较小或无显著性**，表明该集数据泄漏较轻。

### 6.2 NLL 相关性
- 对开源模型（Gemma、Llama、Mistral、StarCoder）：
  - **低 NLL（模型更熟悉）样本在变形后性能下降最强**；
  - Spearman 相关在 0.16–0.40 之间（显著或接近显著），媒介效应强度。  
  → 确认**记忆化样本对变形更敏感**，进一步印证数据泄漏。

### 6.3 不同变形类型影响
- 最敏感的变换：**NestElseIf（嵌套替换）**，在多模型上显著；  
- 结构与命名混合变换（如 NestElseIf + Rename）尤其破坏性强；  
- Gemma‑2‑27B 对单独命名变换也很脆弱，说明其依赖表层标识符；
- Llama‑3.1 与 StarCoder2 均表现普遍性能下降但缺乏显著差异。

### 6.4 综合结论
> “MT + NLL” 联合使用提供了**更强、更可靠的数据泄漏诊断能力**。  
> 变形测试不仅暴露记忆化现象，也能验证模型的真实泛化能力。
