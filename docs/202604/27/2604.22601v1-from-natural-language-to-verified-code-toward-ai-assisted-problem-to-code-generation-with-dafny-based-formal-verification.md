## 摘要
大型语言模型（LLMs）在自动化软件工程中展现出潜力，但它们生成代码的正确性常常被错误或幻觉性内容所削弱。为了确保模型的正确性，形式验证要求 LLMs 在生成实现逻辑的同时，合成能够被数学验证器证明正确的形式化规范。然而，从非正式的自然语言到精确的形式化规范的转换仍是一项艰难的任务。我们的工作通过提供 NaturalLanguage2VerifiedCode (NL2VC)-60 数据集来解决这一问题：该数据集包含 60 个复杂的算法问题。我们在七个开放权重的 LLMs 上，针对 11 个随机选取的问题集进行评估，并采用分层提示策略：无上下文提示、提供结构锚点的函数签名提示，以及利用 Dafny 验证器迭代反馈的自修复提示。为解决“空洞验证”（模型通过生成平凡规范来满足验证器）的现象，我们集成了 uDebug 平台以确保功能验证。结果表明，无上下文提示几乎全面失败，而结构签名和自修复迭代显著改善了性能。具体而言，Gemma 4-31B 达到了 90.91％ 的验证成功率，而 GPT-OSS 120B 在签名引导反馈下从 0 提升至 81.82％。这些发现表明，形式验证现在对于开放权重的 LLMs 已可实现，它们可作为有效的助手来合成复杂注释并促进高保障软件开发。

## Abstract
Large Language Models (LLMs) show promise in automated software engineering, yet their guarantee of correctness is frequently undermined by erroneous or hallucinated code. To enforce model honesty, formal verification requires LLMs to synthesize implementation logic alongside formal specifications that are subsequently proven correct by a mathematical verifier. However, the transition from informal natural language to precise formal specification remains an arduous task. Our work addresses this by providing the NaturalLanguage2VerifiedCode (NL2VC)-60 dataset: a collection of 60 complex algorithmic problems. We evaluate 11 randomly selected problem sets across seven open-weight LLMs using a tiered prompting strategy: contextless prompts, signature prompts providing structural anchors, and self-healing prompts utilizing iterative feedback from the Dafny verifier. To address vacuous verification, where models satisfy verifiers with trivial specifications, we integrate the uDebug platform to ensure functional validation. Our results show that while contextless prompting leads to near-universal failure, structural signatures and iterative self-healing facilitate a dramatic performance turnaround. Specifically, Gemma 4-31B achieved a 90.91\% verification success rate, while GPT-OSS 120B rose from zero to 81.82\% success with signature-guided feedback. These findings indicate that formal verification is now attainable for open-weight LLMs, which serve as effective apprentices for synthesizing complex annotations and facilitating high-assurance software development.

## 一、核心问题与研究背景

- **研究动机**  
  大型语言模型（LLMs）在自动化软件工程领域展示出广泛潜力，但生成代码的**逻辑正确性与安全性**常常受幻觉（hallucination）与逻辑错误影响。  
  当前 LLM 生成的代码多数依赖测试驱动验证，而**缺乏数学证明保证**，难以确保功能正确性和通用可靠性。

- **核心研究问题**  
  如何将自然语言描述（非正式需求）自动转换为**形式可验证的源代码与规格说明（Formal Specification）**，确保生成结果经形式方法严格验证。  
  本研究旨在桥接 AI 自动合成与经典形式化验证（尤其是 Dafny 系统）之间的鸿沟，从而实现**从问题描述到已验证代码的自动生成**。

- **选用 Dafny 的理由**  
  Dafny 是微软开发的**验证感知语言**，集成了“设计契约式”的前后置条件与不变式支持，并通过 Z3 SMT 求解器实现自动证明。  
  但 Dafny 在公共数据中的比例极少（GitHub仅约 779 个仓库），因此在 LLM 训练中是**数据稀缺语言**，为验证 LLM 泛化能力提供理想环境。

## 二、方法论与技术框架

- **核心思想**  
  提出一个从自然语言到正式验证代码的 AI 工作流，包含：
  1. **NaturalLanguage2VerifiedCode (NL2VC-60)** 数据集：  
     包含 60 个自编的复杂算法类问题，每个问题均附验证通过的 Dafny 实现及测试用例。
  2. **三层提示策略（Tiered Prompting Strategy）**：  
     - *Contextless Prompt*：仅输入自然语言任务说明。  
     - *Signature Prompt*：附加 Dafny 函数签名以提供结构锚点。  
     - *Self-Healing Prompt*：在验证器反馈失败时，自动将错误日志回馈给模型迭代修复（模拟人类开发者调试过程）。
  3. **两层验证机制**：
     - 形式验证（Formal Layer）：由 Dafny + Z3 SMT 保证逻辑证明成立。  
     - 功能验证（Functional Layer）：引入 **uDebug 平台** 的社区强力测试用例，杜绝“空洞验证”（即模型以平凡规范通过验证器的现象）。

- **流程概述（文字算法）**
  1. 输入自然语言任务。  
  2. 模型生成初版 Dafny 代码。  
  3. 由 Dafny 验证器检验逻辑一致性。  
  4. 若失败，读取验证器输出，作为 *Self-Healing Prompt* 的迭代输入。  
  5. 重复至最多 10 次或验证通过。  
  6. 对成功代码，进一步在 uDebug 测试集验证功能正确性。

- **关键技术与指标**
  - 使用 *verify@k* 与 *functional@k* 指标衡量成功率。  
  - 温度参数（Temperature）调优测试范围\[0, 0.8\]，分析创造性与逻辑稳定性对验证率的影响。  
  - 错误分类（Error Taxonomy）：语法错误、语义/类型错误、验证错误。

## 六、主要结论与发现

1. **无上下文提示失败普遍**：绝大多数模型 verify@5=0%。  
   - 仅 Gemma 4‑31B（54.55%）和 Codestral‑22B（27.27%）具有限度成功。  
2. **结构签名极大改善**：  
   - GPT‑OSS‑120B 从 0% 升至 63.64%。  
   - Qwen 3.5‑9B 达 72.73%（最佳）。  
   - 表明“形式函数签名”是生成可验证程序的关键。  
3. **自修复验证机制显著提升**：  
   - Gemma 4‑31B 在无签名修复下达 90.91%。  
   - GPT‑OSS‑120B 在签名引导修复下提升至 81.82%。  
   - 自修复通过读取编译器错误补齐不变式和证明条件。  
4. **错误分布规律**：  
   - Contextless 下语法错误主导（>80%）。  
   - 有签名后语义/验证错误成为瓶颈。  
   - 自修复将大部分语义错误转化为已验证代码。  
5. **uDebug 验证功能层有效防止“空洞验证”**，过滤掉仅逻辑一致但语义错误的程序。
