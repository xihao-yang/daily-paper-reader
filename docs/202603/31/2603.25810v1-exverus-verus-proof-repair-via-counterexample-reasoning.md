## 摘要
大型语言模型（LLMs）在自动化形式化验证方面展现出有前景的成果。然而，现有方法将证明生成视为针对源代码的静态端到端预测，依赖有限的验证器反馈，且无法获取具体的程序行为。我们提出了 EXVERUS —— 一个基于反例引导的框架，使 LLM 能够利用反例提供的行为反馈进行证明推理。当证明失败时，EXVERUS 会自动生成并验证反例，然后引导 LLM 将这些反例推广为归纳不变量，以阻止这些失败。我们的评估结果表明，EXVERUS 在证明准确性、鲁棒性和令牌效率方面显著优于现有的基于提示的 Verus 证明生成器。

## Abstract
Large Language Models (LLMs) have shown promising results in automating formal verification. However, existing approaches treat proof generation as a static, end-to-end prediction over source code, relying on limited verifier feedback and lacking access to concrete program behaviors. We present EXVERUS, a counterexample-guided framework that enables LLMs to reason about proofs using behavioral feedback via counterexamples. When a proof fails, EXVERUS automatically generates and validates counterexamples, and then guides the LLM to generalize them into inductive invariants to block these failures. Our evaluation shows that EXVERUS significantly improves proof accuracy, robustness, and token efficiency over the state-of-the-art prompting-based Verus proof generator.

  - 大型语言模型（LLMs）在自动化形式化验证（尤其是 Rust 生态中的 Verus 工具）中表现出潜力，因其生成的证明可以由验证器形式检查。

  - 借鉴传统形式验证中 **反例生成与引导**（counterexample-guided refinement）思想，为 LLM 提供可操作的行为反馈，使修复过程更具针对性与可推广性。
  - 解决 Verus 中反例提取困难的技术瓶颈（SMT 降级导致语义丢失、量词求解不完整等）。

## 2. 方法论（核心思想与技术细节）
### 核心思想
- 在 LLM 驱动的证明生成与修复流程中引入 **源代码级反例生成与验证**。
- 在失败时，合成针对特定错误的高层次 SMT 查询，绕开 Verus 后端低层复杂 VC，直接生成可读、可验证的反例。
- 通过反例分析，归纳出通用失败模式，选择合适的变异策略（mutator），生成候选修复并排序。

### 工作流程（文字化算法）
- **初始证明生成**：沿用 AutoVerus 初始阶段提示模板生成初始 proof annotations。
- **迭代修复**：
  1. **Error提取与优先级排序**  
     从验证器日志抽取错误信息，决定优先修复对象。
  2. **反例生成（CEX GEN）**  
     - 将 proof annotations 翻译为 Z3Py 源代码级约束。
     - 为关键变量设定具体小范围数值，避免量词，最大化 SAT 成功率。
     - 多次生成以获取多个具有代表性的反例。
  3. **反例验证（VALIDATE CEX）**  
     - 对循环不变量错误执行二次验证：在 loop 前后分别检查 invariant，区分 InvFailFront / InvFailEnd。
  4. **错误分类（Error Triage）**  
     - LLM 分析反例可达性：可达 → wrong_fact；不可达 → too_weak；其他类型 → other。
  5. **修复生成（MUT VAL REPAIR）**  
     - 根据 verdict 使用专有 mutator 做替换/加强/其他调整。
     - 提示中明确包含反例与原因，激发 LLM 针对性修改。
  6. **候选排序与验证（RANK TOP）**  
     - 循环不变量类错误按阻塞反例数量排序，非不变量类按子目标验证数排序。
- **结束条件**：验证成功或迭代次数达到上限（默认 10）。

## 6. 主要结论与发现
- EXVERUS 在所有主要基准上的成功率均显著高于 AutoVerus，尤其高难度基准（LCBench, HumanEval）提升可达 2 倍。
- 鲁棒性强，对混淆任务成功率 >73%，显著优于 AutoVerus (<40%)。
- 成本与时间效率显著提高：平均每任务 $0.04（比 AutoVerus 低 4.25×），速度快 4×。
- 多个反例比单一反例更能引导稳定修复；验证模块能有效筛选可用修复。
- 在循环不变量推导瓶颈中，反例推理有效弥补粗粒度错误信息的不足。

## 7. 优点

  - 多反例模式识别与分类，精确选用修复策略（mutator）。
